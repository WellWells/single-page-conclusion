電腦科學術語的「考古學」 🧐

探索 "Core Dump", "Debug", "Patch", "Print" 等常見術語背後，那些源於硬體演進的有趣歷史故事。

📜 核心摘要

許多現今在電腦科學中廣泛使用的術語，其根源可追溯至早期電腦的硬體限制與操作實務。從必須手動修補的打孔卡，到硬體中發現的真實「蟲子」，這些術語的演變忠實反映了技術的快速發展，從具體的物理操作演變為今日抽象的軟體概念。

💾 什麼是「核心轉儲 (Core Dump)」？

這個詞源於 1950 年代廣泛使用的磁芯記憶體 (Magnetic Core Memory)。這種記憶體由許多微小的磁環（稱為 "cores"）網格組成，每個磁環透過磁化方向（順時針或逆時針）來儲存一個位元（0 或 1）。

當程式崩潰時（例如發生「區段錯誤」），作業系統會將當時隨機存取記憶體 (Random Access Memory, RAM) 的完整狀態複製並儲存到一個檔案中，以便後續分析。由於當時的主記憶體就是由這些 "cores" 組成，這個過程很自然地被稱為 「轉儲核心 (dumping the core)」。

🐛 程式「除錯 (Debugging)」的由來

雖然 "Bug"（蟲子）一詞用來形容缺陷在工程領域（如愛迪生的時代）早已存在，但它在電腦科學中的普及，歸功於電腦先驅葛麗絲·霍普 (Grace Hopper)。

1947 年，她在哈佛大學操作 Mark II 電腦時，系統發生故障。團隊追查後，發現一隻飛蛾被困在繼電器（一種早期的電子開關）中。操作員將這隻蛾取出，並貼在日誌本上，註記：「發現了第一個實際的蟲子 (First actual case of bug being found)」。這個雙關語笑話廣為流傳，使 "Debugging"（除錯）成為修復程式問題的代名詞。

🩹 為何修補軟體稱為「修補程式 (Patching)」？

在螢幕和鍵盤普及之前，程式和資料是透過打孔卡 (Punched Cards) 或打孔紙帶 (Punched Tapes) 輸入的。卡片上的「孔洞」代表二進位的 1，「沒有孔洞」則代表 0。

如果打錯了一個孔（即不小心打了 1），要如何將它變回 0 呢？操作員會使用一種快速的「駭客」方法：他們會拿一小片專用膠帶，字面意義上地「修補 (patch)」那個不需要的孔洞，讓讀取器將其辨識為 0。這個物理上的修補動作，演變至今成為發布軟體修正檔的術語。

🖨️ 為何在螢幕上顯示文字叫做「列印 (Print)」？

答案來自電傳打字機 (Teletypes, TTY)。在 1950 至 1970 年代，這些設備是與電腦互動的主要方式。它們本質上是帶有鍵盤的印表機。

當電腦需要輸出資訊時，它會向 TTY 發送訊號，TTY 隨後會實際地將字元「列印 (print)」在紙上。因此，用於「向使用者輸出資訊」的函式很自然地被命名為 `print`。即使後來 TTY 被螢幕（即「終端機」）取代，這個術語因為實用性與歷史慣性而被保留了下來。

🖥️ 「終端 (Terminal)」與「控制台 (Console)」的演進

TTY 也解釋了 美國資訊交換標準碼 (American Standard Code for Information Interchange, ASCII) 表中前 32 個字元（稱為「控制字元」）的由來。它們不是可顯示的字元，而是觸發 TTY 機械動作的指令，例如：

  • 歸位 (Carriage Return, CR) (字元 13): 將列印頭移回行首。
  • 換行 (Line Feed, LF) (字元 10): 將紙張向上滾動一行。

後來，「啞終端機 (Dumb Terminals)」用螢幕取代了紙張，但仍依賴這些控制字元來移動「游標」（Cursor，此詞源於計算尺上的滑動標記）。

至於「控制台 (Console)」，這個詞最初指的是早期大型主機上布滿開關和指示燈的巨大物理操作面板。當 TTY 和終端機取代了這些面板，成為控制系統的主要介面時，它們也繼承了 "Console" 這個名稱。

🏁 總結

這些電腦科學術語的起源，不僅是工程師的懷舊趣聞，它們更清晰地展示了技術的演進脈絡。

從需要物理接觸和機械操作的硬體時代，到現今高度抽象化的軟體世界，這些詞彙如同活化石，提醒著我們每一行程式碼背後，都承載著數十年的創新與迭代。
