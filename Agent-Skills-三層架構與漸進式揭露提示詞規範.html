🎯 什麼是 Agent Skills？

Agent Skills 是 Anthropic 於 2025 年 12 月 18 日正式發布的開放標準，使其與 MCP（Model Context Protocol，模型上下文協定） 一樣，朝著通用跨平台規範的方向發展。 官方甚至提供了開箱即用的 Pre-built Skills（如 Word, Excel, PowerPoint, PDF 處理），讓開發者能直接享用這些強大能力。

一句話總結：Agent Skills 是 一種帶目錄的說明書，或者說是 漸進式揭露提示詞的機制。它將提示詞分成三個層次，實現按需載入，從而大幅降低 Token 消耗與提示詞的複雜度。

🏗️ 三層架構設計原理

Agent Skills 的核心設計理念是 漸進式揭露（Progressive Disclosure），透過分層載入機制，僅在必要時才將完整內容加入 AI 的上下文視窗。

📑

Level 1: 中繼資料層

Metadata Layer

  • • 類比：書籍的目錄
  • • 包含 Skill 名稱與描述
  • • 定義呼叫時機（Trigger）
  • • 必定載入至上下文
  • • 佔用 Token 量極少
📝

Level 2: 指令層

Instructions Layer

  • • 類比：書籍的正文
  • • 詳細的任務執行指令
  • • 使用 Markdown 格式編寫
  • • 按需載入：選擇 Skill 後載入
  • • 可包含範例與規則
📦

Level 3: 資源層

Resources Layer

  • • 類比：書籍的附錄
  • • 可執行腳本（scripts/）
  • • 參考文件（references/）
  • • 圖片等資源（assets/）
  • • 按需載入：指令需要時載入

📄 SKILL.md 檔案結構詳解

每個 Skill 必須包含一個 SKILL.md 定義檔案（注意：SKILL 必須大寫，副檔名 md 為小寫）。此檔案分為兩個主要區塊：

1

中繼資料區塊（Frontmatter）

使用六個橫線 --- 包裹，包含 name 與 description 欄位

2

指令區塊（Body）

使用 Markdown 格式編寫詳細的執行指令與規則

💡 SKILL.md 範例

---
name: 字幕轉 Markdown
description: 當使用者提供 SRT 字幕檔案時，
  將其轉換為 Markdown 格式筆記
---

# 指令說明

你是一個專業的字幕文本處理助手。

## 任務
1. 將 SRT 字幕檔案轉換為 Markdown 筆記
2. **禁止任何刪減、摘要或省略**
3. 必須保留所有文字內容
4. 在關鍵位置插入螢幕截圖佔位符

## 輸出格式
```markdown
![螢幕截圖](screenshot_001.png)
```

🔄 Agent Skills 呼叫流程

理解 Agent Skills 的運作流程是掌握其優勢的關鍵。Agent 支援兩種呼叫模式：

顯式呼叫 (Explicit Invocation)

使用者在提示詞中明確指定要使用的 Skill。

  • • 使用 /skills Slash Command
  • • 輸入 $ 提及特定 Skill
  • • 適用於：明確知道需要哪個工具時

隱式呼叫 (Implicit Invocation)

Agent 根據對話內容自動判斷並啟用合適的 Skill。

  • • 基於 Skill 的 description 進行語意匹配
  • • 自動載入相關指令與資源
  • • 適用於：自然對話場景

以下是完整的自動呼叫判斷機制：

👤
STEP 1

使用者發送提問至 Agent

🔍
STEP 2

掃描所有 SKILL.md 提取中繼資料形成技能清單

🧠
STEP 3

AI 判斷是否需要使用 Skill 及選擇哪一個

📥
STEP 4

載入指令層與必要的資源層內容

✨
STEP 5

綜合所有資訊產生完整回答

💡 關鍵優勢：由於中繼資料都比較簡短，步驟 2 佔用的模型上下文非常少。只有當 AI 選擇使用特定 Skill 後，才會載入詳細指令，體現了按需載入、漸進式揭露的設計模式。

⚡ 資源層進階應用

資源層使 Agent Skills 具備強大的擴展能力。透過腳本執行與參考文件，可實現複雜的自動化流程。

🧩 Agent Ecosystem: 核心元件定位分析

在 Claude 的 Agent 生態系中，Skills、Prompts、Projects、Subagents 與 MCP 各司其職。了解它們的差異與合作方式，是構建強大 Agent 工作流的關鍵。

核心功能程序性知識 (SOP)即時指令背景知識庫任務委派與執行工具與資料連接
持久性跨對話/跨專案單次對話 (短暫)專案內持久跨 Session持續連接

🆚 Skills vs Prompts

Prompts (提示詞)

適用於：「總結這篇文章」、「讓語氣專業一點」。反應式、短暫、用於當下。

Skills (技能)

適用於：「依據 OWASP 標準審查程式碼」、「生成格式化的執行摘要」。主動式、持久、可重複使用。

💡 判斷準則：若你發現自己不斷重複輸入相同的 Prompt，就該把它做成 Skill。

🆚 Skills vs Projects

Projects (專案)

告訴 Agent "What (背景資訊)"。例如：產品規格書、市場研究報告、競品分析。

Skills (技能)

教會 Agent "How (如何做)"。例如：如何撰寫技術文件、如何進行程式碼審查。

💡 合作方式：Projects 提供知識庫，Skills 提供處理這些知識的方法。

🆚 Skills vs Subagents

Subagents (子代理)

像是「專職專員」。擁有獨立的上下文、權限與工具配置。適合獨立執行任務。

Skills (技能)

像是「培訓教材」。是可移植的專業知識。任何 Agent 或 Subagent 都可以學習並使用 Skills。

💡 高級用法：建立一個 Subagent，並讓它配備特定的 Skills 來增強能力。

🆚 Skills vs MCP

MCP (模型上下文協定)

負責 "Connect (連接)"。讓 Agent 能「碰到」外部資料與工具 (讀取 DB、操作 GitHub)。

Skills (技能)

負責 "Teach (教學)"。教導 Agent 該「如何使用」這些資料與工具。

💡 從屬關係：MCP 給予手腳 (能力)，Skills 給予大腦 (SOP)。先有 MCP 連接，再用 Skills 指導操作。

🤝 Skills 與 MCP 合作實戰

Skills 與 MCP 並非互斥關係，而是可以互補合作。Skills 負責提示詞揭露，MCP 負責具體的工具呼叫，發揮各自優勢。

📋 合作範例：自動上傳至 GitHub

   create_repository 工具建立
3. 使用 create_or_update_file 
   工具上傳產出的文件至 GitHub

// 在提示詞中明確指定工具名稱

🛡️ 安全性與使用建議

⚠️ 安全性考量 (Security Considerations)

Agent Skills 具有執行程式碼與讀取檔案的能力，因此必須像安裝軟體一樣謹慎對待。

  • ❗ 來源審計：僅使用來自可信來源的 Skills。
  • ❗ 程式碼檢查：務必檢查 scripts/ 目錄下的腳本內容，防止惡意程式碼執行（如資料外洩、破壞系統）。
  • ❗ 敏感資料：避免讓 Skills 接觸生產環境的敏感憑證與資料。

⚖️ Skills vs Custom Instructions

兩者皆可用於指導 Agent，但適用場景不同：

Custom Instructions (通用指令)

適用於幾乎所有任務都需要的規則。

例：「總是使用繁體中文」、「Python 程式碼需符合 PEP8」。

Agent Skills (專項技能)

適用於特定且詳細的任務流程。

例：「除錯 GitHub Actions 失敗」、「將數據轉換為圖表」。
