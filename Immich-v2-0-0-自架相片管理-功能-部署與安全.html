技術架構

採用 Docker 容器化的微服務架構。後端使用 Nest.js，前端網頁使用 SvelteKit，行動應用則使用 Flutter，確保高效能與跨平台一致性。

部署指南 (Docker Compose)

官方強烈推薦使用 Docker Compose 進行部署，這極大簡化了安裝流程。請確保您的伺服器滿足基本硬體需求（建議 4+ 核心, 6GB+ RAM），並將資料庫檔案儲存在 SSD 上。

  1. 下載官方 docker-compose.yml 和 example.env。
  2. 將 example.env 重命名為 .env。
  3. 編輯 .env 檔案，設定 UPLOAD_LOCATION（儲存路徑）和時區。
  4. 執行 docker compose up -d 啟動服務。
  5. 瀏覽 http://<IP>:2283 註冊管理員帳號。

安全性：VPN 還是公網？

對於自架服務，安全性是最高考量。Immich 處理的是您最私密的個人相片，直接暴露在公網上具有風險。

✅ 建議方案 (最高安全)

使用 VPN (如 WireGuard 或 Tailscale)： 將 Immich 服務保留在您的內部網路 (LAN) 中，不對外開放任何連接埠。當您在外時，透過 VPN 連線回家中網路，即可安全存取服務。

  • 優點：幾乎零攻擊面，資料不經公網。
  • 缺點：行動裝置需常時連線 VPN (可能稍耗電)。

⚠️ 替代方案 (需謹慎)

使用反向代理 (如 Nginx, Caddy)： 透過反向代理將 Immich 暴露於公網，並強制使用 HTTPS (SSL)。

  • 優點：方便存取，無需 VPN，體驗流暢。
  • 缺點：服務直接暴露，需即時更新 Immich 並配置防火牆 (WAF)、Fail2Ban 等防護措施。

結論與建議

Immich 是當前自架領域最接近「完美」的 Google Photos 替代品，尤其在行動應用體驗上遙遙領先。 對於不畏懼 Docker 的技術愛好者，強烈推薦使用。 對於希望建立私有家庭雲的使用者，只要有人能負責維護，它也是絕佳選擇。 但對於完全非技術背景的使用者，商業雲端服務的零維護成本仍是更現實的考量。
