深入解析 OOBE（Out-of-Box Experience，開箱即用體驗）階段繞過微軟帳戶強制登入的多種技術方案，涵蓋審核模式、命令列操作、登錄檔修改及 PE（Preinstallation Environment，預先安裝環境）安裝等進階技巧

📋 重點摘要

🔒 封鎖現況

微軟在最新測試版本中移除了傳統的 bypassnro.cmd 腳本，且自 2025 年下半年起，針對「僅限本機帳戶」的繞過機制進行了更系統性的封鎖（部分無效化）。

透過 Windows 強大的向下相容性，仍有多種歷史遺留方法可在預覽版中使用本機帳戶安裝系統。

🎯 適用情境

新機測試、避免微軟帳戶名稱作為使用者名稱、無網路環境安裝、批次部署等多種使用場景。

🔍 什麼是 OOBE（開箱即用體驗）？

OOBE 的全稱是 Out-of-Box Experience（中文譯為：開箱即用體驗）。它是 Windows 作業系統安裝過程中的最後一個階段，也就是您在重灌系統、或是購買新電腦第一次開機時，所看到的那個藍色漸層、引導您進行個人化設定的畫面。

1. OOBE 的核心功能

  • 地區與語系設定：選擇所在國家及慣用輸入法。
  • 接受授權條款：確認同意微軟合約 (EULA)。
  • 網路連接：Windows 11 目前強制要求連線。
  • 帳戶建立：設定名稱並強制登入微軟帳戶 (MSA)。
  • 隱私設定：選擇定位、診斷資料等選項。

2. 技術底層邏輯

OOBE 雖然在 C:\Windows\System32\oobe 有核心元件，但現代 Windows 11 大量利用 CloudExperienceHost (ms-cxh) 呈現網頁式前端。這意味著微軟能更容易在流程與 App 層面封鎖繞過手段，Shift+F10 僅是介入的一個窗口。

3. 為什麼大家想要「繞過」OOBE？

解除強制連網

在無驅動或無網路環境下避免卡死。

使用本機帳戶

確保隱私並自訂正確的使用者資料夾名稱。

跳過硬體檢查

在舊裝置上強制完成系統設定。

💡 小知識： 後續教學中提到的指令 msoobe，其實就是手動觸發 OOBE 處理程序的執行檔。當我們輸入指令時，就是強制系統重新載入這個設定精靈。

🔧 方法一：Audit Mode（審核模式）快速進入桌面

Audit Mode 是品牌電腦製造商用於預先安裝驅動程式、調整設定及載入預裝軟體的專用環境。此方法可讓您跳過 OOBE 流程，快速進入桌面環境以進行測試或維護。

📝 操作步驟

1

在 OOBE 初始設定畫面（越早越好，建議在選擇地區頁面）中，按下快速鍵 Ctrl + Shift + F3

（若無反應，請嘗試 Fn + Ctrl + Shift + F3）

2

系統將自動重啟並登入至 內建管理員帳戶 (Built-in Administrator)

3

進入桌面後將自動顯示 Sysprep 工具視窗

⚠️ 關鍵限制與注意事項

  • 非永久性繞過：此模式僅跳過「互動式 OOBE 介面」進入桌面。一旦執行 Sysprep 將系統封裝（Reseal）並回到 OOBE，部分 oobeSystem 階段的設定腳本可能仍會執行。
  • 帳戶權限差異：您使用的是系統內建的 Administrator 帳戶，其安全權限模型與一般使用者不同，且部分功能（如指紋辨識、特定 UWP 應用）可能受限。
  • Sysprep 視窗：在未退出 Audit Mode 前，每次開機都會彈出 Sysprep 視窗，這是正常現象。
  • 建議用途：適合硬體驗機、軟體預裝測試。若目標是「建立長期使用的本機帳戶」，建議搭配其他方法使用。

⌨️ 方法二：BypassNRO 登錄檔修改（備用方案）

⚠️ 關鍵前置條件

使用此方法時，務必物理斷網（拔除網路線或關閉 Wi-Fi）。若系統偵測到網路連接，即使修改了登錄值，也極高機率會強制導向微軟帳戶登入。此外，隨著版本更新（如 24H2 後期 build），微軟正持續封鎖此漏洞，成功率可能浮動。

此方法原理是透過修改 Windows 登錄檔，嘗試強制開啟隱藏的「我沒有網際網路」按鈕。

📝 快速指令操作（推薦）

1

在 OOBE 畫面按下 Shift + F10 開啟命令提示字元

2

直接輸入下方指令（設定值並強制寫入）：

reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\OOBE /v BypassNRO /t REG_DWORD /d 1 /f
3

輸入重啟指令：

shutdown /r /t 0

🚫 關於舊版 bypassnro.cmd

舊版教學常提到的「從 System32 複製 bypassnro.cmd」方法（原方法 A），在 Build 26120.3653 後已被微軟移除該檔案，且舊版檔案在新系統也可能無法執行，因此推薦直接使用上述指令。

💻 方法三：使用 Net User 命令預建帳戶（實驗性質）

🧪 實驗性/應急方案

此方法透過命令列直接建立使用者，在部分舊版本或特定情境下有效。但需注意：「建立帳戶」不等於「跳過 OOBE 強制登入流程」。在較新的 Windows 11 版本中，即使建立了本機帳戶，OOBE 仍可能卡在強制登入微軟帳戶的畫面，因此建議將此視為備援手段。

OOBE 的核心功能之一即為建立使用者帳戶。透過命令列工具，可以手動完成 OOBE 原本執行的帳戶建立流程，直接繞過連網驗證步驟。

📝 操作步驟

1

在 OOBE 畫面按下 Shift + F10 開啟命令提示字元

（Dell 電腦使用者請嘗試 Shift + Fn + F10）

2

輸入以下指令建立新使用者（參數與密碼為選填）：

net user "使用者名稱" "密碼" /add

💡 建議設定密碼以確保帳戶安全。

3

將新使用者加入系統管理員群組：

net localgroup administrators "使用者名稱" /add
4

執行 OOBE 完成流程：

cd oobe msoobe

執行 msoobe 後系統通常會自動重啟。若無反應，可手動輸入 shutdown /r /t 0。

🚀 進階參數（可選）

若需更完整的帳戶控制，可使用以下延伸指令：

# 確保帳戶啟用
net user "使用者名稱" /active:yes
# 設定帳戶永不過期
net user "使用者名稱" /expires:never

重啟後將直接以管理員身分進入桌面，無需再經歷微軟帳戶登入流程。

🎛️ 方法四：OOBE JavaScript 控制台（高技術性）

此方法利用 OOBE 原始碼中隱藏的開發者控制台，直接呼叫舊版帳戶設定介面。這是一個技術性極強的繞過方式，但需注意微軟已在 2025 年底開始測試封鎖此類 local-only 路徑，未來可能會隨更新失效。

📝 詳細操作步驟

1

開始設定並停留在「次要鍵盤設定」畫面

選擇地區與主鍵盤後，當出現「是否新增第二種鍵盤設定？」畫面時，請勿點擊跳過。

2

開啟開發者控制台

按下 Ctrl + Shift + J。畫面將變暗，左上角出現 > 提示符號。

3

輸入重啟指令

輸入以下指令（大小寫敏感）：

WinJS.Application.restart("ms-cxh://LOCALONLY")

💡 技巧：輸入 WinJS.A 按 Tab 鍵自動完成，再輸入 res 按 Tab 鍵自動完成。

4

執行並退出

按下 Enter 執行，接著按 Esc 退出控制台（若 Esc 無效，請點選畫面任意處重新聚焦控制台後再試）。

5

完成本機帳戶設定

畫面將刷新並顯示 Windows 10 風格的本機帳戶設定頁面。輸入使用者名稱與密碼（可留空）即可完成。

🛠️ 疑難排解 (Troubleshooting)

  • 控制台無法開啟：嘗試點選畫面空白處，確保焦點在 setup box 之外再重試快速鍵。若在微軟帳戶登入畫面（輸入 Email 處），控制台會被封鎖。
  • 卡在載入畫面：若輸入指令後無反應，可能是 URI（括號內文字）輸入錯誤。請長按電源鍵強制重啟電腦。
  • 已錯過鍵盤設定畫面：若此時卡在「網路設定」或「解鎖微軟體驗」畫面，建議重啟電腦從頭開始；或嘗試在「為此裝置命名」畫面開啟控制台。

💡 技術原理

此指令觸發了 js/errorHandler.js 中原本用於錯誤處理的隱藏按鈕事件 (_localAccountButton)，強制 OOBE 重啟並進入簡易版的本機帳戶設定流程。

🌐 方法五：連網狀態下隱藏微軟帳戶登入（官方機制）

💎 企業級穩定技術

此方法利用微軟為企業部署設計的 Unattended Setup (回應檔) 參數。這雖然是社群發現的登錄檔對應解法，但因為屬於官方支援的部署邏輯，穩定性通常高於單純的腳本繞過。

此方法本質上是手動寫入微軟 Unattend (回應檔) 用於自動化部署的設定值。透過修改 HideOnlineAccountScreens 項目，模擬企業部署環境，讓系統在 OOBE 階段隱藏線上帳戶登入畫面。

📝 快速指令操作（推薦）

1

在 OOBE 畫面按下 Shift + F10 開啟命令提示字元

2

一次輸入下方兩個指令（或合併輸入）：

# 隱藏線上帳戶登入頁面
reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\OOBE /v HideOnlineAccountScreens /t REG_DWORD /d 1 /f
# 跳過無線網路設定頁面（可選，增加成功率）
reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\OOBE /v HideWirelessSetupInOOBE /t REG_DWORD /d 1 /f
3

輸入 shutdown /r /t 0 重新啟動系統

📌 方案優勢

  • 全版本支援：適用於 Windows 11 家用版 (Home) 及專業版 (Pro)。
  • 安全性高：由於是官方預留的自動化安裝參數，相較於 BypassNRO 更不容易因為版本更新而失效。
  • 連網更新：允許系統在設定過程中下載必要的累積更新。

💾 進階方案：製作預設跳過連網的 USB 安裝碟

若需頻繁為多台電腦安裝系統，可透過專業工具預先製作含跳過連網設定的 USB 安裝碟，無需在每台電腦上手動執行命令。

🛠️ 使用 Rufus 製作安裝碟

  1. 1. 從微軟官方網站下載 Windows 11 ISO 映像檔
  2. 2. 開啟 Rufus，選擇目標 USB 隨身碟
  3. 3. 點選「選擇」載入 ISO 映像檔
  4. 4. 點選「開始」後，勾選以下選項：
  • ✅ 移除 Windows 安裝限制
  • ✅ 跳過線上帳戶要求
  • ✅ 建立本機帳戶
  • ✅ 停用常見的自動將裝置加密 (BitLocker)
  • ✅ 關閉隱私權相關設定

⚠️ 安裝時請選擇「舊版安裝程式」選項以確保跳過連網功能生效

🎯 使用 Ventoy 多重開機碟

  1. 1. 使用 Ventoy 將 USB 製作為多重開機碟
  2. 2. 直接將 Windows ISO 映像檔拖曳至 USB
  3. 3. 開機後進入 Ventoy 選單
  4. 4. 進入設定頁面，確認已啟用：
  • ✅ 跳過系統安裝限制
  • ✅ 跳過連網需求

💡 Tip: 請確認 Ventoy 的外掛設定 (Global Control Plugin) 中已正確啟用 VTOY_WIN11_BYPASS_NRO，且安裝時必須暫時移除網路。

🖥️ 進階方案：WinPE + Unattend.xml 自動化安裝

🏢 企業級部署標準

這是專業 IT 管理員與裝機愛好者最常用的方法。透過 WinPE (Windows Preinstallation Environment) 環境搭配 Unattend.xml (回應檔)，可以實現完全自動化安裝，徹底跳過 OOBE 並預建帳戶。

�️ 核心原理與工具

回應檔（應對檔）可預先定義：使用者名稱/密碼、地區語言、自動過關 EULA、磁碟分割設定等，讓系統在首次啟動時即處於可用狀態。

  • 推薦工具：使用 Schneegans Unattend Generator 或官方的 Windows System Image Manager (SIM) 產生 autounattend.xml。
  • 裝機方案：將 ISO 放進 USB，並將產生好的 xml 檔放在 USB 根目錄，安裝程式會自動偵測並執行。

⚠️ 關鍵限制與風險

  • 24H2/25H2 相容性：最新預覽版可能會忽略部分 WinPE 階段的 Unattend 設定（如鍵盤設定），可能仍需手動微調。
  • 工具純淨度：部分第三方 PE 工具（如某些「一鍵裝機」軟體）可能夾帶廣告或修改系統元件。建議使用官方 WinPE 或知名純淨版 (如微 PE)。

⚠️ 🔓 危險技巧：永久停留審核模式 (Audit Mode Persistence)

⛔ 極高風險警告

此方法雖正確，但僅建議用於臨時測試或特定實驗環境。長期在此狀態下使用系統會導致安全性與穩定性嚴重受損，絕不適合日常辦公或娛樂使用。

其基本原理是透過替換 sysprep.exe 執行檔，防止系統執行重設流程並返回 OOBE，強制系統在每次重啟後都停留在此特殊環境。

📝 操作流程簡述

  1. 1. 在審核模式桌面開啟工作管理員，定位 sysprep.exe 位置。
  2. 2. 獲取該檔案的擁有者權限（更改為 Administrators 群組並賦予完全控制）。
  3. 3. 備份後刪除原檔，並將一個無害程式（如 notepad.exe）更名為 sysprep.exe 存回原處。

☢️ 潛在負面影響

  • ❌ UAC 失效：系統將處於內建管理員狀態，部分安全性防禦失效，且部分應用程式可能無法正確請求權限。
  • ❌ 功能受限：指紋登入、Windows Hello、部分 UWP 應用程式可能無法正常執行。
  • ❌ 更新異常：系統更新與特定的驅動程式安裝可能出現攔截或錯誤。
