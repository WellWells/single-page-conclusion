第二章：核心定位深度對比

在 LLM（大語言模型）時代，GPU 的價值不再僅僅是 FP32 運算能力，而是其「吞吐資訊」的能力。理解這兩類硬體的本質差異，是決定部署架構的第一步。

消費級 RTX 系列

  • 設計目標：單工卓越與極致 CP 值

    追求單點運算效能，針對遊戲、影像編輯進行優化。硬體相對單純，主打「隨插即用」，不需要昂貴的機房環境即可運作。

  • 瓶頸在於「資訊往返」

    由於 GDDR 頻寬較窄且不支援 NVLink，在處理 LLM 這種需要反覆將大量權重輸入核心的任務時，輸出速度（Token/s）會明顯受限。

  • 驅動與擴充

    著重於 Game Ready 與 Studio 驅動，不支援多卡記憶體池合併，適合單卡載入 7B–14B 等級模型進行輕量微調。

資料中心 GPU 系列

  • 設計目標：極致吞吐量與規模化

    專為 24/7 叢集運算設計。其核心價值在於「頻寬」與「容量」，能同時處理數百個請求而不會因為記憶體存取而塞車。

  • LLM 的核心指標：VRAM 與 頻寬

    搭載 HBM3e 記憶體，提供數 TB/s 的頻寬，這直接決定了 Token 的輸出速度；而超大 VRAM 則決定了能否載入 70B 甚至更大模型。

  • NVLink 超能力

    支援 NVLink 5，能合併數張 GPU 的 VRAM 形成統一記憶體池，這是處理超大規模 LLM 推論的唯一選擇。

明確差異化對照：LLM 需要什麼？

1. 記憶體容量 (VRAM)

決定模型的「居住空間」

如果 VRAM 不夠大，模型根本「裝不下」。資料中心卡透過大容量與 NVLink，讓 70B 滿精度模型得以載入，而消費型卡大多只能執行「量化後」的縮小版。

2. 記憶體頻寬 (Bandwidth)

決定輸出的「流暢度」

LLM 屬於「自迴歸」解碼，需要反覆搬運權重。頻寬越高，每秒輸出的 Token 越多。這就是為什麼 H100 (HBM) 的速度遠超 4090 (GDDR) 的主因。

3. 部署複雜度

決定「進入門檻」

消費型顯卡只要插上主機板即可工作；資料中心 GPU 則需要昂貴的機架冷卻系統、高額授權費與專業維運團隊。對於個人開發者，消費級顯卡依然是首選。

核心結論：如果您在機殼內擠入四張 RTX 4090，雖然 VRAM 疊加了，但 PCIe 的傳輸瓶頸會讓多卡溝通極慢，在 LLM 表現上永遠無法與單台具備 NVLink 的 H100 相提並論。

VRAM 對消費型 LLM 的絕對優勢

在個人開發環境中，VRAM 容量 (Video RAM) 就是一切。它代表了 GPU 能夠即時存放模型數據的「工作記憶體」。

  • 決定模型規模：24GB (4090) 勉強能跑 30B 模型，32GB (5090) 搭配量化或 mixed precision 可實務上運行 34B 模型。
  • LoRA 微調必備：微調時需要同時存放模型權重與梯度數據，VRAM 越大，訓練批次 (Batch Size) 才能越高，速度才快。
  • Context Window：對話紀錄越長，佔用的 VRAM 越多。大 VRAM 讓您能使用 32k 甚至 128k 的長文本。

TFLOPS 運算能力：肌肉量不代表一切

TFLOPS (Tera Floating-point Operations Per Second) 代表 GPU 每秒能進行多少兆次浮點運算。

這就像是一輛車的「引擎馬力」。雖然馬力大很重要，但在 LLM 推論中，如果路徑 (頻寬) 太窄，馬力根本無法全開。這解釋了為什麼 4090 運算效能高，但在大型模型輸出速度上仍會輸給運算效能稍低但頻寬更高的專業卡。

第四章：記憶體架構與雙重頻寬謎題

TL;DR：記憶體頻寬與 NVLink 頻寬到底差在哪？

1. 記憶體頻寬 (Memory Bandwidth) 是「對內」的速度：單一 GPU 從自己的 VRAM 讀取權重到運算核心的速度。它決定了單張顯示卡的效能極限與出 Token 的速度。
2. NVLink 頻寬 (Interconnect Bandwidth) 是「對外」的速度：多張 GPU 之間互相傳遞資料的速度。當模型大到單張卡裝不下，必須切分給多張卡運算時，如果 GPU 之間傳資料太慢，運算效能再強也得等。NVLink 確保了 GPU 「組隊」時不會互相造成效能受限。

1. 記憶體頻寬 (Memory Bandwidth)

Internal: GPU Core ↔ VRAM

這是「家裡的走廊」。它負責將數據從顯卡上的 VRAM 搬到運算核心。對於 LLM 推論 (Inference) 來說，這項指標決定了 Token 輸出的流暢度。資料中心使用 HBM，頻寬是消費型 GDDR 的數倍。

2. NVLink 頻寬 (Interconnect Bandwidth)

External: GPU ↔ GPU

這是「 GPU 之間的跨海大橋」。資料中心需要同時跑數千億參數的模型，單張卡的 VRAM 裝不下，必須拆分到多張卡。NVLink 讓多張卡能以極低延遲共享數據，讓它們「看起來像一張超大顯卡」。

為什麼只有資料中心需要 NVLink？

解決模型規模問題

當您要跑 175B (GPT-3 等級) 模型，需要 8 張 H100 串聯。如果沒有 NVLink，數據在 GPU 之間傳輸會卡在慢速的 PCIe 匯流排上，導致效能崩潰。

分散式訓練 (Training)

訓練大模型需要同步所有 GPU 的權重梯度。NVLink 的雙向高頻寬是確保數萬顆 GPU 同步運作、不產生等待時間的唯一解方。

跨海大橋的進化：NVLink 5 vs NVLink 6

專為 Rubin 設計的 NVLink 6 實現了頻寬再次翻倍，延續了 NVIDIA 將整個資料中心視為「單一巨型晶片」的 Jensen's Law 設計哲學。相對於 PCIe 6.0，NVLink 6 提供了近 14 倍的傳輸優勢。

個人家用評估：該買 RTX 5090 還是 GB10？

如果你的首要目標是「在家穩定跑 70B 等級大模型」，選 GB10；如果你還想兼顧遊戲、影像、Windows 日常使用，且主要跑 7B–34B 模型，用 RTX 5090 比較適合。

先講結論：兩張圖幫你選

你該選 RTX 5090 的情況

你需要一台「同時是遊戲機／工作站＋AI 卡」的主機（Windows 為主）。主要跑的模型在 7B–34B，偶爾玩 70B（可以接受激進量化、雲端輔助）。想要最快的單卡推論效能（7B–13B token/s 非常兇），也會玩 SD/影片生成。願意自己處理高瓦數 PSU、散熱與機殼空間。

優點
  • 可以裝進你現有/未來的桌機，當一般 GPU 用。
  • 跑 7B–13B 模型，vLLM / Ollama 效能幾乎是消費級天花板。
  • 日後可以配 RunPod / Lambda 租 H100 專門處理 70B 訓練或大型專案。
缺點
  • 32GB VRAM 單卡 無法舒服撐起 70B Q4，只能玩重壓縮 Q2/Q3，品質與穩定性打折。
  • 雙卡 5090 在台北家用電力上已逼近 110V/15A 上限，且沒有 NVLink，多卡 LLM 擴展效率有限。
你該選 ASUS Ascent GX10 (GB10) 的情況

你最看重的是：在家、一次到位能長期穩定跑 70B 甚至更大模型。接受「這台只幹 AI 工作站，不拿來打電動」，Linux OK。你想要的是「不用煩惱 PSU / 機箱 / 散熱 / 配線，一接電就能跑 LLM」。開發重心是 LLM / RAG / 企業級 PoC，而不是 3D 或遊戲渲染。

優點
  • 128GB 統一記憶體，可以「正面硬接」70B FP16 或更高階量化模型。
  • 整機只吃 240W，110V 家用插座完全沒壓力，噪音與散熱也比雙 5090 主機好控制。
  • 企業級 DGX 系列簡化版，對之後如果你要接企業案（私有 LLM 部署）很有展示力。
缺點
  • 幾乎只能當「AI 盒子」，不適合作為日常桌機／遊戲 PC（Linux + 專用軟體堆疊）。
  • GPU 架構是 GB10 平台，軟體生態相對封閉，很多消費級調校工具、遊戲完全派不上用場。
  • 價位雖跟 5090 主機總價接近，但「彈性」顯著低於自己組的 5090 平台。

2. 更大型但非「個人級」的方案

(C) 真正資料中心 GPU：H100 / B200 / Rubin
  • 單台 B200 伺服器建置成本多在千萬等級新台幣，且受配額、授權、維運成本影響極大。

  • 對「家用」來說幾乎是不可行選項，只存在於：你自己就是小型 data center，或公司買機器你有 root 權限能當實驗室用。

3. 不買更大卡，用雲端補齊的路線

這其實是很多 4090 / 5090 玩家在 2026 真正採用的路線：

  • 在家：一張 4090 / 5090 跑 7B–34B、做 RAG、工具鏈開發、前後端整合。
  • 雲端：需要訓練或大量跑 70B / 405B 時，上 RunPod / Lambda / Vast「租 H100/B200 幾小時」跑完就收。
優點

不用承擔高額一次性 CAPEX。每月用量不大時，TCO（總持有成本）實際上比你買超高階硬體還低。

缺點

資料跨境傳輸與隱私問題（對企業客戶需做 anonymization 或 on-prem）。有時候 queue / 搶熱門機型比較煩。

第六章：非 NVIDIA 陣營的挑戰與現況

NVIDIA 的霸主地位在 2026 年依然穩固，其最大護城河正是深不可測的 CUDA 生態系。然而，隨著算力需求暴增，其他科技巨頭「殺入市場」的腳步從未停歇。以下是 AMD、Google 以及 Intel 的最新市況，以及現今最強大 AI 模型背後的硬體秘辛。

AMD：最強硬體挑戰者

  • Instinct MI300X & MI325X MI300X (192 GB HBM3) 直接對標 H100；而最新的 MI325X (256 GB HBM3e) 在記憶體容量與頻寬（約 6 TB/s）上更具優勢，目前已獲 Microsoft Azure、Meta、Oracle 等重量級背書。
  • ROCm 生態系與挑戰 ROCm 雖為開放的軟體生態，支援 PyTorch 等框架，但其安裝部署、除錯工具與文檔的成熟度仍落後 CUDA，實務導入成本較高，這也是「可用但不算好用」的主因。

Google：自給自足的算力巨獸

  • TPU v5 / v7 (Ironwood) 不對外單獨販售晶片，僅透過 Google Cloud 提供。專為矩陣與張量運算最佳化，透過數千顆 TPU 建立超大 Pod 叢集。
  • 市場定位 不賣 GPU 卡，而是主打雲端算力服務（Vertex AI）。對超大客戶是很棒的替代來源，但對自建機房的企業則無緣。

Intel：性價比與第二供應商

  • Gaudi 2 & Gaudi 3 Gaudi 2 配備 96 GB HBM，Gaudi 3 升級至 128 GB HBM2e (約 3.7 TB/s 頻寬)。主打每美元算力極高與乙太網互聯。
  • 生態策略 與 Hugging Face、AWS (EC2 UltraClusters) 深度合作以降低開發門檻。算力指標雖不僅 NVIDIA，但在利基市場與成本敏感型客戶中具有吸引力。

頂級 AI 模型都在用什麼硬體？

ChatGPT (OpenAI)

OpenAI 的大型模型主要在 Microsoft Azure 上運行，背後是大規模的 NVIDIA GPU（A100、H100、未來的 Blackwell），高度仰賴 CUDA 最佳化來追求訓練與推論效能。不過近年也開始嘗試導入如 Google TPU 等其他硬體，以降低對單一供應商的依賴。

Gemini (Google)

Gemini 幾乎是 Google 自家 TPU 的展示平台，核心訓練與推論主要建立在 Google Cloud TPU 叢集上，從早期的 v4，發展到針對大模型的 v5p，再到 2025 年推出的第七代 TPU v7（Ironwood）。Google 透過這套自有硬體堆疊，顯著減少對 NVIDIA GPU 的依賴。

Claude (Anthropic)

Anthropic 過去以 AWS 為主力，結合 NVIDIA GPU 與 AWS 自研的 Trainium／Inferentia 晶片。自 2025 年起，又與 Google Cloud 簽下龐大 TPU 長期合約，以支援下一代 Claude 模型訓練與服務。目前策略是 Google TPU、AWS 自研晶片與 NVIDIA GPU 並行，刻意避免被綁死在單一硬體供應商。

總結：對於一般家用或企業自建機房（On-Premise）客戶，NVIDIA 依舊是繞不開的首選；但如果您目光放遠至雲端叢集或身為超大規模的底層開發者，AMD 的 MI300/325 系列與 Google TPU 已經實質殺入戰場，為這個壟斷的市場撕開了一道極具價格與記憶體優勢的破口。
