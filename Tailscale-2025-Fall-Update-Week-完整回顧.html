Tailscale 2025 🚀 Fall Update Week 回顧

更簡單、更智能、更緊密。我們正在讓安全網路變得（以一種好的方式）無聊——橫跨人員、服務與其間的黏合程式碼。

⚙️ 毫不費力的 Tailnet 管理

🖱️ Visual Policy Editor (GA)

目標： 讓 ACL 策略管理不再是 JSON 專家的特權。

解決了什麼： 徹底告別手動編輯 HuJSON 檔案時，因忘記逗號、引號或縮排錯誤而導致的挫敗感。這個 GUI 介面使得管理權限變得直觀，非常適合技術背景較淺的團隊成員（如服務台）或「中等技術」團隊接手管理。它就像一份「即時文件」，無效的選項根本不會出現在下拉選單中。

General Availability

🗂️ Multiple Tailnets (Alpha)

目標： 提供企業級的絕對網路隔離，而無需管理多個組織或 IdP。

解決了什麼： 就像是「Tailnet 的資料夾」。它允許您在單一組織下建立多個完全封閉、互不相通的 Tailnet。這對於需要嚴格區分 Dev、Test、Prod 環境的企業，或是 MSP 服務商為不同客戶管理網路來說，是個改變遊戲規則的功能。它還支援 API 驅動，可建立後銷毀的「暫時性測試網路」。

Open Alpha (Enterprise)

🛡️ 安全的服務連接

📡 Tailscale Services (Beta)

目標： 提供一個內部的、ACL 控制的負載平衡器與反向代理。

解決了什麼： 它將「服務」從「實體主機」中解耦。您可以將三個 Proxmox 節點的 WebUI 放入同一個 Service，實現內部服務的高可用性 (HA)。它也解決了「我不想記住 8080、9090 等各種埠號」的問題，並為內部資源提供了自訂的 MagicDNS 名稱。

Open Beta

🤖 Workload Identity Federation (Beta)

目標： 讓自動化服務（而非人類）安全地存取 Tailnet。

解決了什麼： 徹底告別長期的 API 金鑰。您的 GitHub Actions、AWS Lambda 或 Azure VM 現在可以使用短暫的、有範圍限制的 OIDC Token 自動向 Tailnet 驗證，無需在程式碼庫中儲存任何秘密金鑰，大幅降低管理開銷與安全風險。

Beta

💻 無縫的終端用戶體驗

🌐 Peer Relays (Beta)

目標： 解決 DERP 中繼的效能瓶頸並降低雲端出口成本。

解決了什麼： Tailscale 超過 90% 的連線是 P2P 直連，但剩下 <10% 會 fallback 到 DERP (TCP-in-TCP，效能有限)。Peer Relays 允許您自行部署 UDP 中繼站，在 "Hard NAT" 環境（如 AWS NAT Gateway）中，可提供高達 10 倍的效能提升。它還能讓雲端流量保持在同一區域內，節省高昂的出口 (egress) 費用。

Public Beta

🖥️ Windowed Desktop Client (macOS)

目標： 提供一個功能更豐富的桌面客戶端體驗。

解決了什麼： macOS 客戶端不再只是一個小小的選單列圖示。這個全新的獨立視窗化應用程式提供了足夠的「地產」來容納搜尋、更好的錯誤訊息、連線偵錯以及 Taildrop 等未來功能。這也預示著其他桌面平台（Linux/Windows）將獲得更強大的 UI 體驗。

Feature Preview

🌐 為無縫安全連接而生的平台

🛡️ TSIDP (Tailscale Identity Provider)

目標： 消除 Tailnet 內部應用程式的登入摩擦。

解決了什麼： 實現「零點擊登入」。您已在 Tailnet 上驗證了自己的身份，為什麼還要再次登入 Grafana、Proxmox 或 Audiobookshelf？TSIDP 作為一個輕量級 OIDC 供應商，能將您的 Tailnet 身份無縫傳遞給任何支援 OIDC 的應用程式，徹底告別登入彈窗。

Open Source

🧩 App Capabilities

目標： 將 ACL 的精細控制權從「網路層」擴展到「應用層」。

解決了什麼： 讓開發人員無需 SDK 即可在應用程式內部實現功能級別的權限控制。您的應用程式只需檢查傳入的 HTTP 標頭，就能根據 Tailscale ACL 授予的「能力」，決定用戶是「唯讀」還是「可讀寫」，實現真正的最小權限原則。
