🚀 GitHub Code Quality：AI 世代下的程式碼品質擴展

整合 Copilot 智慧修復與 CodeQL 靜態分析，在 Artificial Intelligence (AI) 加速開發的同時，實現自動化的程式碼體質監測與治理。

🏎️

核心摘要：速度與控制的辯證

講者以 F1 賽車 作為隱喻：賽車手若只有速度而無法過彎，終將失控。在軟體開發中，AI 是加速引擎（Speed），而 程式碼審查（Code Review）則是過彎技巧（Control）。

GitHub 透過整合靜態分析與大型語言模型（LLM），打破過去「速度 vs 品質」的權衡（Trade-off），讓開發者在不犧牲速度的前提下，利用 Copilot Autofix 實現規模化的品質治理。

核心功能架構

雙重檢測引擎

結合 CodeQL 的結構化分析（語法、邏輯漏洞）與 LLM 的語意理解能力。CodeQL 負責精準打擊，AI 負責廣泛覆蓋，互補不足。

Copilot Autofix

秉持「發現即修復」(Found means Fixed)。在 PR 階段不僅標示錯誤，更提供單鍵修復建議。若有 Copilot 授權，更可指派 Coding Agent 批次處理。

治理儀表板

提供組織層級的視覺化報表，追蹤 可靠性 (Reliability) 與 可維護性 (Maintainability) 分數。可設定 Rule Sets 阻擋低品質程式碼合併。

⚡

Quickstart：實戰運作流程

1

啟用與掃描 (Enable & Scan)

啟用 Code Quality 後，系統會針對每一個新 PR 及預設分支 (Default Branch) 進行掃描。初次啟用需等待數分鐘以完成基準掃描。

2

識別高風險項目 (Identify)

進入 Security 標籤頁的 Code Quality 儀表板。利用過濾器鎖定 高嚴重性 (Error/Warning) 的問題。解決這些問題能最顯著提升程式碼健康分數。

3

生成 AI 修復 (Autofix)

點擊問題旁邊的 ✨ Generate fix 按鈕。審查 Copilot 建議的程式碼變更，確認無誤後直接點擊 "Open pull request"。

4

合併與觀測 (Merge & Observe)

合併修復 PR 後，Code Quality 會自動重新掃描。回到儀表板，您應能觀察到 Reliability 與 Maintainability 分數的具體提升。
