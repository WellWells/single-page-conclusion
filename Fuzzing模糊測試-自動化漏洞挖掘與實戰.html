Week 2: Exploit Development

🧪 模糊測試 (Fuzzing)：
讓程式崩潰的藝術

如果不看源始碼，你要怎麼找出程式的漏洞？
答案很簡單：「一直餵它吃奇怪的東西，直到它噎死為止。」
這就是 Fuzzing 的核心精神。本週我們將學習如何自動化這個過程，從應用程式一路炸到 Linux 核心。

🐵

什麼是 Fuzzing？

想像你是一隻坐在電腦前的猴子，任務是找出 Word 的 Bug。你可能會亂敲鍵盤、亂點滑鼠。 傳統的 Fuzzing 就是這隻猴子（隨機亂試）。但現代的 Coverage-Guided Fuzzing（覆蓋率導向）是一隻有智慧的猴子。

🧠 聰明的猴子 (AFL++)

  • 👉 監控程式反應：它會在程式裡裝監視器（插樁 Instrumentation）。
  • 👉 學習路徑：如果輸入 "ABC" 讓程式跑到了新的程式碼區塊（例如進入了 if 判斷式），它會把 "ABC" 記下來。
  • 👉 變異 (Mutation)：下次它會試 "ABC+1"、"ABC%@"，而不是從頭亂試。

🕵️ 隱形的裁判 (Sanitizers)

程式有時候「內傷」了（例如記憶體些微寫錯），但外表看起來還在跑，這時候 Fuzzer 會以為沒事。

AddressSanitizer (ASAN) 就像一個隱形裁判，只要程式偷偷踩到不該踩的記憶體紅線，裁判立刻吹哨判犯規（強制 Crash），讓我們知道這裡有洞。

🛠️ 本週軍火庫：工具比較

🐇

AFL++

通用型重型機槍

用途： 最標準、最流行的工具。適合測一般的二進位檔案（如圖片轉檔工具、影片播放器）。

特點： 透過基因演算法不斷變異輸入，速度快，支援度廣。

🧪

Google FuzzTest

手術刀級精準打擊

用途： 寫在 C++ 單元測試裡面。適合測單一函式或特定邏輯。

特點： 開發者友善。不像 AFL++ 要測整個程式，它可以只測某個小函式會不會崩潰。

🐧

Syzkaller

針對核心的核彈

用途： 專門炸 Linux Kernel（作業系統核心）。

特點： 非常難搞。因為核心一崩潰整台電腦就掛了，所以必須在虛擬機 (QEMU) 裡面跑。它專門找那種「多工處理時才會發生的錯誤」(Race Condition)。

🔥 實戰流程詳解

從準備子彈到引爆的過程

Step 1

準備種子 (Seed Corpus)

Fuzzer 雖然聰明，但如果你給它空檔案，它要花很久才能猜出「這是一個 JPG 圖片」。

正確做法： 去網路上下載一堆「合法、正常」的小圖片（種子）餵給它。它會基於這些正常的圖片去修改（改壞它），這樣更容易深入程式邏輯。
關鍵技巧： Corpus Minimization（去蕪存菁）。如果 100 張圖片跑到的程式路徑都一樣，留 1 張就好，不然浪費時間。

Step 2

撰寫驅動程式 (Harness)

這是最技術性的一步。Fuzzer 產生的是一串記憶體資料，目標程式可能是一個 GUI 軟體。你需要寫一個「轉接頭」。

// 這個轉接頭函數會被 Fuzzer 瘋狂呼叫一秒幾千次 extern "C" int LLVMFuzzerTestOneInput(const uint8_t *data, size_t size) { // 1. 這裡不用讀檔案 (fopen)，直接拿 data // 2. 把 data 餵給目標函式 TargetFunction(data, size); // 3. 記得清理記憶體，不然還沒測到漏洞記憶體就爆了 return 0; }

⚠️ 重點：不要在 Harness 裡面寫 printf 或是讀寫硬碟，那會讓速度變超級慢。我們要的是記憶體內的極速碰撞。

Step 3

崩潰分析 (Crash Analysis)

程式掛了！然後呢？不是每個 Crash 都能變成駭客入侵的漏洞（Exploit）。

  • 高危險 (Exploitable)：你可以控制記憶體寫入的內容，或操控程式跳轉的位置（如 Heap Overflow, UAF）。
  • 低危險 (DoS)：程式只是當機，但駭客沒辦法拿到權限（如 Null Pointer Dereference）。

使用工具 CASR 可以幫你自動分類這些 Crash 嚴不嚴重。

🌍 真實世界的恐怖故事

案例 1：圖片炸彈 (libWebP)

Chrome 和 Firefox 都在用的圖片解析庫。AFL++ 發現如果把圖片裡的霍夫曼表 (Huffman Table) 改壞，程式會算錯記憶體大小，導致堆積溢位 (Heap Overflow)。這意味著只要你在網頁上放一張特製圖片，使用者瀏覽時電腦就被駭了。

案例 2：核心驚魂 (Linux Netfilter)

Syzkaller 發現 Linux 防火牆模組裡有一個競爭條件 (Race Condition)。如果你在極短時間內同時建立又刪除規則，核心會搞混，去存取已經被釋放的記憶體 (Use-After-Free)。這讓普通使用者可以取得 Root 權限。

本週心法總結

Fuzzing 不是魔法，它是一種「暴力美學」。
我們利用電腦不知疲倦的特性，去窮舉人類邏輯想不到的邊界情況。
學會 Fuzzing，你不再是被動等待 Patch 的使用者，而是能主動找出 0-day 漏洞的獵人。
