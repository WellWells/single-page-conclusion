APT36 與 Vibeware 的崛起

以量取勝的 AI 輔助惡意軟體工業化。Bitdefender 以中度信心將此活動關聯至巴基斯坦國家支持的駭客組織 APT36（Transparent Tribe），其主要攻擊目標為印度政府、外交使節與國防機構，其次為阿富汗政府。
威脅不在單一樣本多厲害，而在可近乎每日產出新變種、並行部署多個不同語言植入體。Bitdefender 將此作法描述為『Distributed Denial of Detection』，用來強調以大量多樣樣本耗損防守方分析能量的概念。

執行摘要

Bitdefender 研究指出，APT36（Transparent Tribe）疑似開始採用 AI 輔助的『vibeware』開發模式（又稱為 Vibe-coding，即提示詞寫程式），利用 Nim、Zig、Crystal、Rust、Go 等語言快速產生大量變種，並把 C2 與資料外洩活動隱藏在 Slack、Discord、Google Sheets、Supabase、Firebase 等可信服務中；Bitdefender 以中度信心將此活動關聯至 APT36。

該組織的重點活動為竊取特定資料，其目標鎖定為：軍事人員文件、外交通訊、國家安全與戰略政策文件。

惡意軟體工業化

程式碼品質雖低，例如研究人員發現一個 Go 語言竊密軟體的 C2 伺服器地址竟只是測試用的佔位符 (Placeholder)，導致根本無法回傳資料。但產量極大，這凸顯了其以幾乎每日更新的頻率淹沒目標環境、「只求量不求精」的特徵。

多語言混合 (Polyglot)

利用冷門語言重置傳統資安工具的偵測基準，規避依賴特定特徵碼或已知行為模式的防護機制。

濫用可信服務

將指令與控制 (Command and Control, C2) 流量隱藏於 Slack, Discord, Google Sheets 等合法商業服務中，增加阻斷難度。

核心攻擊概念深度解析

APT36 改變了網路戰的經濟學，不追求完美的單一武器，而是用龐大的數量與多樣性來耗盡防禦者的精力。

分散式阻斷偵測 (Distributed Denial of Detection, DDoD)

這是一種概念上的轉變。攻擊者的目標不再是透過高超的技術繞過安全控制，而是透過自動化生成的海量不同特徵的惡意軟體來耗盡防禦者的遙測與分析能力。

  • 多重植入：受害者通常同時被感染多種不同語言寫成的後門。
  • 彈性存活：防禦者切斷一條路徑（例如封鎖某個 C2 協定），其他使用不同協定的後門依然存活。
  • 事件回應複雜化：資安團隊必須識別並移除多個據點，大幅增加清理成本。

寄生可信服務 (Living Off Trusted Services, LOTS)

相較於傳統的本地寄生二進位檔 (Living Off the Land Binaries, LOLBins)，寄生可信服務 (LOTS，Living off Trusted Sites) 是將惡意活動隱藏在廣泛使用的雲端服務中。LLM 非常擅長生成與這些服務 API 互動的穩定程式碼。

// 攻擊者基礎架構足跡
Payload_Store: Firebase, Supabase, Google Drive
C2_Instructions: Google Sheets Cells, Discord Msg
Data_Exfil: Slack Channels, tmpfiles.org, Google Drive

這使得惡意流量混入企業正常的 HTTPS 流量中，單純封鎖 IP 或網域不再有效，因為企業無法輕易封鎖 Google 或 Slack。

初始存取 (Initial Access)

報告指出，發現 LinkedIn 頁面截圖，顯示攻擊者利用專業社群網路進行目標側寫 (High-value targets)。初始存取與散佈手法包含：

  • 誘餌 PDF： 文件內含履歷圖片及「Download Document」按鈕，點擊後觸發 ZIP 或 ISO 壓縮檔下載，欺騙依賴靜態掃描的設備。
  • LNK 執行後，會在記憶體中無檔案 (fileless) 執行 PowerShell，下載載入器 (如 Warcode 或 NimShellcodeLoader)，再由載入器反射部署 Cobalt Strike 或 Havoc 等進階框架。

突破 ABE 保護

ABE

Google 在 Chrome 127 for Windows 先將 應用程式綁定加密 (App-Bound Encryption, ABE) 用於 cookies，並規劃逐步擴展到其他敏感資料；因此，相關竊資工具的重點已從單純讀取使用者層級加密資料，轉向注入瀏覽器程序以取得解密能力。

攻擊手法 (LuminousCookies)： 透過尋找 chrome.exe、brave.exe 或 edge.exe 處理程序，強制載入惡意 DLL 來冒充合法元件。

1. Create suspended browser process
2. Write payload.dll path to memory
3. CreateRemoteThread(LoadLibrary)
4. Wait for named pipe signal

OPSEC 失誤與歸因混淆

在歸因分析 (Attribution) 中，基礎架構足跡透露了開發者的可能身分與反情報線索：

  • 開發環境關聯： LuminousStealer 的路徑中出現 C:\Users\kumar\.cargo\（Rust 套件管理器目錄），而 LuminousCookies 內的 C:\Users\kumar\ 則間接連結至 SupaServ 的開發環境。
  • 開發者代號 Nightmare： 攻擊者基礎架構與工具鏈中，重複出現用戶名 "Nightmare"，Bitdefender 認為是核心開發或運營人員。
  • Discord 命名： C2 基礎架構所使用的 Discord 伺服器，被命名為韓國知名網漫《我獨自升級》（Solo Leveling）主角 Jinwoo 的「Jinwoo's Server」。

🛡️ 防禦與緩解建議

Vibeware 代表了技術的倒退，但策略的升級。依賴特徵碼的靜態偵測已被淘汰，企業必須採取以下現代防禦措施。

1. 強化行為偵測 (Behavioral)

將偵測重心從檔案執行階段轉向程序行為與通訊基準線 (Baselining)。正常的 chrome.exe 連線至 Discord 是合理的，但若背景執行且未簽章的二進位檔頻繁對這些 API 發起呼叫，即為高風險異常。EDR 應對 DLL 注入、排程工作建立、記憶體內 PowerShell 執行、對 Slack/Discord/Google Sheets/Supabase/Firebase 的異常 API 存取 發出警報。

2. 監控多重植入 (Parallel Implants)

事件回應 (IR) 時應謹記：「發現一個，就要預期還有三個。」 清理單一以 Nim 等較冷門語言編寫的後門時，必須立即針對相同時間點的執行緒與網路活動進行深度掃描，尋找可能潛伏的 Zig 或 Go 等其他語言變種。

3. 建立敵意網路環境

雖然 malware production pipeline 工業化，但後續操作仍多為人工互動，不代表整個入侵鏈已完全自動化。 透過實施動態攻擊面縮減 (ASR) 及部署成熟的 XDR 與 MDR 營運，為攻擊者製造摩擦與不確定性，迫使他們在探測階段暴露行蹤。
