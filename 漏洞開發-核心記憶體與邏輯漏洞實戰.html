第一天：記憶體損壞基礎

Memory Corruption Fundamentals

Stack Buffer Overflow (堆疊緩衝區溢位)

當程式向堆疊（Stack）上的緩衝區寫入超過其容量的資料時發生，導致相鄰記憶體被覆蓋。這可能損壞返回地址（Return Address），允許攻擊者重導程式執行流程。

CASE STUDY: CVE-2024-27130

QNAP QTS/QuTS hero Stack Overflow

漏洞成因： 使用不安全的 strcpy() 複製使用者輸入至固定大小的緩衝區，缺乏長度驗證。
攻擊影響： 遠端程式碼執行（Remote Code Execution, RCE），攻擊者可取得 NAS 系統完全控制權。
修復方式： 替換為邊界檢查函數並增加輸入驗證。

Use-After-Free (UAF)

程式在記憶體釋放後仍繼續使用該指標（Dangling Pointer）。攻擊者可透過堆疊噴射（Heap Spray）或堆疊風水（Heap Feng-Shui）技術控制該記憶體內容。

CASE STUDY: CVE-2024-2883

Chrome ANGLE Use-After-Free

漏洞成因： WebGL 上下文銷毀時，圖形操作仍引用該記憶體。
攻擊影響： 透過惡意網頁觸發 RCE，無需使用者互動。
重要性： 瀏覽器與圖形子系統因複雜的物件生命週期管理，常成為 UAF 的重災區。

Heap Buffer Overflow

寫入超過動態分配（Heap）緩衝區的邊界。通常導致 Heap Metadata 或相鄰物件損壞，當分配器（Allocator）處理時引發崩潰或執行程式碼。

CASE STUDY: CVE-2023-4863

libWebP Heap Buffer Overflow

漏洞成因： BuildHuffmanTable() 處理 Huffman 編碼表時邊界檢查不足。
影響範圍： 廣泛影響 Chrome, Firefox, Android 等數十億設備（供應鏈風險）。

Info Leak & Uninitialized Memory

越界讀取 (OOB Read) 或使用未初始化的記憶體。這類漏洞通常不直接導致 RCE，但對於繞過 KASLR (Kernel Address Space Layout Randomization) 至關重要。

CVE-2024-53108 (Linux AMDGPU)

OOB Read 導致核心記憶體洩漏。

CVE-2024-26581 (Linux Netfilter)

未初始化變數洩漏堆疊資料，配合 CAP_NET_ADMIN 進行提權。

第二天：邏輯漏洞與競爭條件

Logic Vulnerabilities & Race Conditions

Time-of-Check Time-of-Use (TOCTOU)

競爭條件的一種，發生在「檢查條件」與「使用資源」之間存在時間差。攻擊者可在此時間窗口內改變資源狀態（如符號連結替換）。

CVE-2024-26218 (Windows Kernel)

核心在驗證權限後未進行原子操作（Atomic Operation），允許攻擊者在檢查與使用之間修改資源，導致本地提權（LPE）至 SYSTEM。

Double-Fetch (CVE-2023-4155)

核心從使用者空間讀取資料兩次。第一次用於驗證，第二次用於執行。攻擊者在兩次讀取間修改資料，導致驗證失效。案例發生於 Linux KVM AMD SEV。

邏輯與認證繞過

不涉及記憶體損壞，而是程式邏輯設計缺陷。

CVE-2024-0012
Palo Alto PAN-OS Auth Bypass

未經身分驗證的攻擊者可直接繞過登入檢查取得管理員權限，完全無需利用記憶體漏洞。

Arbitrary Write (Write-What-Where)

如 CVE-2024-21338 (Windows AppLocker Driver)，透過 IOCTL 處理不當，攻擊者可將任意值寫入核心任意位置（例如覆蓋 PreviousMode）以繞過安全檢查。

第三天：類型混淆與整數漏洞

Type Confusion, Integer Overflows & Parser Bugs

Type Confusion (類型混淆)

當程式將物件視為與其實際類型不同的類型進行處理時發生。這在 C++ 轉型或 JIT 編譯器（如 V8）優化中很常見。

實戰案例：CVE-2024-7971 (V8 TurboFan)

  • 背景： V8 引擎的 TurboFan 優化器錯誤地假設了陣列元素的類型。
  • 攻擊： 透過構造混合類型陣列觸發 JIT 優化錯誤，混淆 Tagged Pointer 與 SMI (Small Integer)。
  • 後果： 透過偽造物件（fakeobj）獲得任意讀寫能力，最終繞過 V8 Sandbox 執行程式碼。

Integer Underflow & Overflow

整數運算超出變數儲存範圍。特別危險的是下溢 (Underflow)，例如 0 - 1 在無號整數中變成巨大的正數。

CVE-2024-38063 (Windows TCP/IP)

IPv6 封包處理時發生整數下溢，導致計算出的緩衝區大小極小，但在複製資料時使用原始長度，引發 Heap Overflow 與 RCE。

Signed-to-Unsigned Conversion

有號整數（負數）被強制轉型為無號整數時，會變成巨大的正數。

CVE-2024-47606 (GStreamer)

MP4 解析器中，有號整數計算下溢成負數，轉型為 size_t (unsigned) 後導致記憶體分配錯誤與堆疊溢位。

第四天：格式化字串與注入攻擊

Format Strings & Parser Vulnerabilities

Format String Bugs

當使用者輸入被直接作為格式化函數（如 printf, syslog）的參數時發生。

// Vulnerable
printf(user_input);

// Safe
printf("%s", user_input);
CVE-2023-35086 (ASUS Router)

攻擊者利用 %p 洩漏堆疊地址繞過 ASLR，再利用 %n 寫入任意地址，覆蓋返回地址達成 RCE。

Parser Complexities

檔案解析器（Archive, Document, Media）因格式複雜且需處理不信任輸入，是高風險目標。

  • WinRAR Logic Flaw (CVE-2023-38831)

    利用檔名欺騙（同名的資料夾與檔案），使使用者點擊看似安全的 PDF 時執行惡意執行檔。

  • Word RTF Heap Overflow (CVE-2023-21716)

    字體表（Font Table）解析時邊界檢查失敗，導致 Heap Corruption 與 RCE。

第五天：驅動程式與啟動安全

Kernel Drivers, Filesystems & Boot Security

IOCTL Handler Bugs

驅動程式與使用者空間溝通的橋樑。常見漏洞包括大小混淆與指標驗證失敗。

CVE-2023-21768 (Windows AFD.sys)

輸入緩衝區大小驗證邏輯錯誤，導致核心池（Kernel Pool）越界寫入與本地提權。

BYOVD

Bring Your Own Vulnerable Driver。攻擊者載入合法的、但有漏洞的簽名驅動程式，利用其介面進入核心，繞過驅動程式簽章強制（DSE）。

Lazarus Group Technique

利用舊版 Dell/ASUS 驅動程式關閉 EDR 或 PatchGuard。

FS & Boot Security

檔案系統邏輯與啟動過程的漏洞可導致持久化控制或容器逃逸。

Dirty Pipe (CVE-2022-0847) 未正確初始化 Pipe 標誌，允許覆蓋唯讀檔案（如 /etc/passwd）。
Secure Boot Bypass (CVE-2025-47827) 簽章驗證缺陷允許載入惡意核心。
