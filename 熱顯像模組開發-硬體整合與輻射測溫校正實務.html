紅外線測溫原理與技術基礎

Infrared Sensing Principle & Technical Foundation

非冷卻型長波紅外線（LWIR, Long-Wave Infrared）熱成像感測器屬於先進的微測輻射計系統。其測溫原理結合了物理學的熱輻射定律與微機電 (MEMS) / CMOS 半導體製程技術。

1

物理基礎：熱輻射吸收

  • 紅外線波段： 偵測 8 到 14 微米 (μm) 的長波紅外線 (LWIR)，這是人體與室溫物體發射熱輻射的主要波段。
  • 能量轉換： 輻射透過鏡頭聚焦到像素陣列 (FPA) 上，像素吸收能量產生溫升。
  • 真空封裝 (WLVP)： 晶圓級真空封裝技術阻斷熱傳導與對流，確保像素能敏銳保留微小熱能。
2

光電轉換：溫差發電效應

  • 混合架構： 採用微熱輻射熱計 (Microbolometer) 與熱電堆 (Thermopile) 混合技術。
  • 電壓產生： 像素吸收輻射後產生的微幅溫升會透過熱電效應轉化為微小電壓。
  • 相對測量： 產生的電壓取決於「視野物體」與「晶片本身 (Die Temp)」的溫差。
3

訊號處理與數位化

  • 類比前端 (AFE)： 處理晶片將像素產生的微弱電壓訊號進行高精度放大與濾波。
  • 數位轉換 (ADC)： 放大後的訊號被轉換為數位數值 (Raw ADC counts)。
  • 溫度參考： 必須同時讀取感測器內部的封裝溫度 ($T_{die}$) 作為計算基準。
4

溫度計算與校正

  • 逐像素校正： 利用 Flash 內的 Look-up Tables 消除像素間的不均勻性。
  • 輻射測溫公式 (Radiometry)： 結合基座溫度、物體發射率 (Emissivity)、靈敏度因子與偏移量進行補償。
  • 最終輸出： 以 deci-Kelvin (dK) 格式透過 SPI 輸出，供主機端直接轉換。

總結： 物體發出紅外線 → 像素吸收升溫 → 產生相對電壓差 → 數位化 → 加上晶片溫度並經過參數補償 → 輸出目標溫度。

協同處理器與通訊介面

Companion IC Role & Interface Selection

處理晶片核心職責

協同處理晶片（Companion IC）負責處理底層控制與原始資料運算，為 Host 主機提供標準化介面。其完整資料流水線（Pipeline）包含：

  • ▸原始資料擷取：從感測器讀取類比訊號並進行 ADC 轉換。
  • ▸像素級校正 (Per-pixel Calibration)：修正個別像素的增益與偏移誤差。
  • ▸壞點修正 (BPC)：偵測失效像素並利用鄰近像素進行數值補償。
  • ▸溫度轉換 (Temp Conversion)：將原始資料轉換為絕對溫度（Kelvin）。
  • ▸雜訊抑制 (Denoising)：執行時間與空間濾波演算法。

通訊介面配置策略

處理晶片支援多種主機通訊模式，通常於硬體重置時透過特定腳位的電位狀態來決定：

USB 介面模式

當 MODE Pin 拉高（Pull-up）時啟用。適用於 PC 或高階嵌入式系統，直接模擬為 USB 裝置進行資料傳輸。

SPI + I2C 組合模式

當 MODE Pin 拉低（Pull-down）時啟用。此為 MCU 開發最常用模式：
• I2C (Control)： 用於發送指令、讀取狀態暫存器（Master-Slave 架構）。
• SPI (Data)： 專用於高速熱影像資料（Thermal Data Frame）傳輸。

控制流程與軟體實作

Software Logic & Control Flow

1. 硬體介面與接腳規劃

系統透過偵測模式切換腳位 (MODE) 的電位來決定通訊方式：

SPI + I2C 模式 (MCU 開發)

MODE 接腳下拉 (Pull-down)。這是嵌入式系統開發的標準模式，分別處理控制指令與高頻資料。

USB 模式 (PC/Android)

MODE 接腳上拉 (Pull-up)。處理晶片模擬為 USB CDC 裝置（虛擬 COM Port），使用 ASCII 字串通訊。

2. 嵌入式系統控制邏輯 (SPI + I2C)

A. 初始化與啟動

  • 重置程序： 將 nRESET 接腳拉低至少 32µs 以進行硬體重置。
  • 等待就緒： 正常啟動約需 1.5 秒。若接新感測器模組，首次啟動可能需 3~5 秒。
  • 狀態輪詢： 讀取暫存器 0xB6，監控 BOOTING_UP 標誌位。

B. 參數配置 (I2C)

  • 啟動串流： 寫入 0xB1 選擇連續模式 (0x02) 或單張模式 (0x01)。
  • 設定幀率 (FPS)： 透過 0xB4 暫存器設定除數。
  • 增益範圍控制： 寫入 0xB9，可選自動增益或指定高中低檔（測溫上限隨之調整）。

C. 資料讀取 (SPI)

  • DR 中斷： 當資料就緒，DATA_READY 接腳會由低變高。
  • SPI 模式： Master 設為 Mode 0 (CPOL=0, CPHA=0)。
  • 資料幀構造： 讀取完整 Data Frame（包含 Header 與 Pixel Data）。

3. 資料解析與轉換

溫度轉換： 讀取的 16-bit 原始資料通常為 deci-Kelvin (dK) 格式。

Example: 3005 dK = 300.5 K = 27.35 °C

可透過 0x31 暫存器將輸出單位切換為攝氏或華氏。

核心迴圈虛擬碼 
Init(I2C, SPI);
Write_Reg(0xB1, 0x02); // Enable Continuous Mode
while(true) {
  if (Pin_DataReady == HIGH) {
    Read_SPI_Buffer(data_array);
    Process_Temperature(data_array);
  }
}

🎛️ 暫存器控制詳解 (Register Control Map)

本傳感器系列提供豐富的 I2C 暫存器，可精細調整系統行為。以下為四大關鍵控制類別：

1. 影像擷取控制

Acquisition Control
FRAME_MODE (0xB1) R/W

核心開關。Bit 1 啟動連續串流；Bit 7 在「溫度資料」與「補償Raw」間切換。

FRAME_RATE (0xB4) R/W

設定除數 (Divider) 以降低 FPS。適合慢速 MCU 或增強訊號積分時間。

2. 溫度測量校正

Radiometry & Calibration
EMISSIVITY (0xCA) 0~100

發射率補償。預設 0x5F (95%)。測量金屬或特殊材質時務必調整。

SENS_FACTOR (0xC2) Float

保護窗穿透率補償 (如 1.23x)。還原被阻擋的熱輻射能量。

OFFSET_CORR (0xCB) +/-

全域溫度偏移。消除環境熱或外殼自發熱造成的截距誤差。

GAIN_CTRL (0xB9) Mode

高增益 (<120°C)、低增益 (>400°C) 或 Auto 切換。

3. 濾波與演算法

Quality Optimization
FILTER_CTRL (0xD0) R/W

濾波總開關。啟用時間濾波 (Temporal)、移動平均 (Rolling Avg) 等。

STARK_CTRL (0x20) Adv

進階時空濾波 (Spatio-Temporal)。平衡去噪與動態殘影，適合即時監控。

4. 系統管理

Power & Status
SLEEP_MODE (0xB5) Time

設定讀取後的休眠時長。節能關鍵。

USER_FLASH (0xD8) NVM

存取內建 Flash，可用於儲存使用者自定義的校正參數或序號。

�

開發建議： 初期只需專注於 0xB1 (開畫面) 與 0xCA/0xC2 (調準度)。濾波器 (0xD0) 建議在確定原始資料正確後再開啟。

進階熱管理與機構設計

Thermal Physics & Mechanical Integration

熱傳導挑戰

熱感測器的準確度極易受 PCB 熱應力影響。主要設計目標是阻斷熱源傳導至感測器，確保感測器處於均溫狀態（Isothermal State）。

PCB 布局規範

  • 銅箔切口 (Cut-outs)： 移除感測器周圍的接地層與電源層，增加熱阻。
  • 元件布局： 遠離高功率元件（如 MCU、LDO）。
  • 物理隔離： 在模組與 PCB 間使用絕緣墊片。

外殼材質物理特性

外殼材質的熱導率 (Thermal Conductivity, k) 決定了溫度梯度的分布，直接影響測量誤差。

鋁合金 (Aluminum) 推薦

高熱導率促使外殼溫度分布均勻，消除局部熱點，使感測器能快速達到熱平衡，穩態誤差極小。

FR4 / 塑膠 需謹慎

低熱導率易導致溫度梯度，產生難以補償的動態誤差。

保護視窗 (Protective Window)

引入視窗會帶來兩大挑戰，需透過軟體與機構共同解決：

  1. 訊號衰減 (Attenuation)： 視窗吸收紅外線，導致讀數偏低。
  2. 自發輻射 (Self-emission)： 視窗自身溫度會成為干擾熱源。
材料建議： 必須使用對 LWIR 波段透明的材料，如 矽 (Si) 或 鍺 (Ge)，並施加 AR 鍍膜。

訊號處理與濾波演算法

Signal Processing & STARK Filter

時域 vs 空域濾波器

濾波器類型 工作原理 優缺點分析
時間濾波 (Temporal) 計算單一像素在連續多幀的平均值。 優： 靜態場景極佳
缺： 動態場景產生鬼影 (Ghosting)
空間濾波 (Spatial) 計算單幀內鄰近像素的數值（如中值濾波）。 優： 去除椒鹽雜訊
缺： 可能模糊細微特徵
STARK 濾波器 Spatio-Temporal Advanced Rolling Kernel
結合時域與空域資訊。		 最佳平衡方案。但在快速動態場景中，雜訊可能被誤判為運動模糊。

色彩映射核心：Min-Max Scaling

將溫度資料視覺化時，必須使用最小-最大特徵縮放，將感測器溫度範圍映射至 0-255 的顏色索引。

Index = (T_pixel - T_min) * 255
---------------------------
(T_max - T_min)

* T_min 與 T_max 可透過「自動增益」動態調整，或「手動鎖定」以維持背景一致性。

黑體測量理論與邏輯

Blackbody Measurement Theory & Principles

1. 什麼是黑體測量？

在熱成像技術中，黑體 (Blackbody) 是一個絕對標準的參考熱輻射源。它能設定在特定溫度並發出已知且穩定的紅外輻射。

核心定義

理想黑體具有 發射率 $\varepsilon = 1.0$ (100%)，能完美發射熱輻射而不反射環境雜訊。它是驗證校正參數 (Calibration) 的唯一真理標準。

工程用途

用於修正感測器因組裝公差或加裝保護窗 (Window) 後產生的讀數誤差。透過測量已知溫度的黑體，逆向推導出系統所需的補償參數。

2. 測量原理：溫差檢測法

熱傳感器並非直接測量絕對溫度，而是測量「目標物體」與「感測器晶片」之間的溫差能量。

  • ▸ 黑體發射： 產生 8-14 $\mu m$ 長波紅外線。
  • ▸ 差分感測： 像素電壓 $\propto (T_{target}^4 - T_{die}^4)$。
  • ▸ 訊號重建： 處理晶片結合 Raw Data 與 $T_{die}$ 計算主要讀數。
誤差修正模型
$T_{final} \approx \sigma \cdot (T_{raw} + \text{Offset})$
• $\sigma$ (Sensitivity): 修正斜率 (如保護窗衰減)
• Offset: 修正截距 (如環境熱干擾)

3. 背後的物理理論

輻射發射率 (Emissivity)

基於 斯蒂芬-波茲曼定律 ($P = \varepsilon \sigma_{sb} T^4$)。若使用非理想黑體 (如 $\varepsilon=0.97$)，必須在晶片參數中設定對應發射率，否則高溫段會產生誤差。

線性兩點校正 (Two-point)

雖然熱輻射是非線性的，但在工作區間內可視為線性。利用高溫黑體與低溫黑體的兩點資料，計算出實際響應斜率：

$\sigma_{new} = \frac{T_{readout} - T_{die}}{T_{blackbody} - T_{die}}$

光學穿透率 (Transmission)

矽保護窗會吸收能量。理論上，修正係數與穿透率成反比 ($\sigma \approx 1 / \tau_{av}$)。這是實作中調整 Sensitivity Factor 的物理依據。

關鍵校正參數與衰減補償

Calibration Parameters & Attenuation Compensation

保護視窗之訊號衰減補償 (σ)

當在感測器前加裝 保護窗 (Protective Window) 時，材質（如矽 Si 或鍺 Ge）會阻擋部分紅外線輻射，導致訊號變弱並使測得溫度低於實際值。此時必須調整 靈敏度因子 (Sensitivity Factor, σ) 來進行補償。

方法 1 簡易計算法 (基於穿透率)

若已知保護窗在 LWIR (8-14 μm) 波段的平均穿透率 (Transmission Rate, $\tau_{av}$)，可用此公式：

σ = 1.0 / τ_av

範例： 穿透率為 73% (0.73) 的矽晶圓。

計算： σ = 1.0 / 0.73 ≈ 1.37

方法 2 實驗校正法 (基於黑體測量)

最精確的方法，利用標準黑體輻射源消除製造公差：

  1. 將黑體設定於高溫（建議 80°C）以獲得強訊號。
  2. 將裝有保護窗的模組對準黑體。
  3. 調整 σ 數值，直到感測器讀數與黑體溫度一致。
  4. 進階： 可計算「無視窗」與「有視窗」之資料斜率比值。

如何寫入設定？

暫存器名稱

SENSITIVITY_FACTOR

暫存器地址

0xC2

數值格式

浮點數因子 × 100 轉整數

範例：若因子為 1.23 → 寫入值 = 1.23 * 100 = 123 (0x7B)

發射率校正

依據目標材質調整。地址：0xCA，預設 0x5F (95%)。正確設定此值對絕對溫度準確性至關重要。

物理意義： 描述物體發射紅外能量相對於黑體的能力。

溫度偏移 (Offset)

消除環境或自發熱造成的截距誤差。地址：0xCB。

建議： 在校正靈敏度 (σ) 後，應針對室溫環境進行一次 Offset 校正以確保精度。
T_final = T_raw * σ + Offset

保護視窗整合注意事項

• 物理距離： 建議視窗與鏡頭保持 1-2 mm 距離，以減少熱干擾。
• 自發輻射： 視窗本身受熱後的輻射會導致基準偏移，需透過 Offset 補償。

開發注意事項與整合重點

Development Precautions & Integration Highlights

🌡️ 熱管理與機構設計

  • PCB 熱隔離： 建議在感測器下方區域進行 Cut-out 處理，阻斷來自 MCU 或電源晶片的熱傳導。
  • 機構間隙： 模組金屬面應避免直接接觸受熱不均的外殼，建議保留 1-2mm 空氣層 作為絕緣。
  • 均溫狀態： 設計應確保感測器晶片與鏡頭座維持熱平衡，快速的外殼降溫或升溫會導致動態偏差。

⚡ 高品質電源設計

  • 超低雜訊 LDO： 推薦使用輸出雜訊 < 6.5 µVRMS 的 LDO 為感測器供電（如 LP5907）。
  • 走線規劃： 電源走線應儘量短且寬，減少 IR drop。電壓波動會直接轉化為影像中的條紋雜訊或測溫誤差。

💻 軟體與暫存器控制

  • 揮發性設定： 校正暫存器（如 Sensitivity, Offset）在斷電後會重置，主機應在每次啟動時重新寫入。
  • 增益切換限制： 自動增益切換瞬間會產生短暫條紋。若手動切換增益，建議暫停擷取並等待 1s 穩定。
  • 啟動延遲： 系統重置後至少需等待 1.5s 才能讀取資料，切勿在狀態暫存器就緒前通訊。

⏹️ 系統限制與操作

  • 無機械快門 (Shutterless)： 長時間運作若出現殘影，需透過軟體演算法或短暫重新校正補償。
  • 通訊時序： 若 I2C/SPI 輪詢速度過快且未考慮 DATA_READY 訊號，可能導致資料撕裂或幀率下降。

📚 工程術語表 (Glossary)

LWIR (Long-Wave Infrared) 長波紅外線，波長 8-14 μm。矽 (Si) 與鍺 (Ge) 視窗對此波段具有良好透明度。
NETD (Noise Equivalent Temperature Difference) 雜訊等效溫差。衡量感測器能辨識的最小溫度差異，數值越低靈敏度越高。
FPA (Focal Plane Array) 焦平面陣列。由多個感測像素組成的二維陣列，位於鏡頭焦點處。
PTAT (Proportional to Absolute Temperature) 與絕對溫度成正比的數值，用於追蹤感測器內部溫度變化。
BPC (Bad Pixel Correction) 壞點修正。利用鄰近正常像素的溫度估算值來替換壞點讀數。
ADC (Analog-to-Digital Converter) 類比數位轉換器。將連續的類比訊號轉換為離散的數位數值。
Isothermal State (均溫狀態) 感測器晶片與鏡頭座溫度一致的理想狀態，可將溫度梯度誤差降至最低。
Emissivity (發射率) 物體發射紅外輻射的能力比值，理想黑體為 1.0，人體皮膚約 0.98。
WLVP (Wafer-Level Vacuum Packaging) 晶圓級真空封裝。阻斷熱傳導與對流，提升感測器熱靈敏度的關鍵技術。

💡 工程師常見問答 (Technical FAQ)

Q: 增益控制 (Gain Control) 如何影響測量範圍與靈敏度？ ▼

透過增益暫存器控制模組範圍：

  • 高增益 (High Gain)： 靈敏度 (NETD) 最佳，雜訊最低，但測溫上限較低（約 120°C）。
  • 低增益 (Low Gain)： 犧牲部分靈敏度，換取高達 400°C 以上的寬廣測溫範圍。
  • 自動增益 (Auto)： 系統依場景溫度自動切換，但在臨界點可能發生頻繁跳動。
Q: 若必須將感測器置於發熱的塑膠外殼內，有何緩解建議？ ▼

這是不利於測量的環境。建議措施：

  1. 極大化距離：將感測器儘可能遠離熱源元件。
  2. 增加熱阻：在 PCB 上感測器周圍進行挖空 (Cut-out)，並在感測器與 PCB 間加裝絕緣墊。
  3. 空氣間隙：確保感測器背板與塑膠外殼不直接接觸，保留空氣層作為絕緣。
Q: 為什麼讀取的溫度總是偏低？如何校正？ ▼

溫度偏低通常有以下原因：

  • 保護窗衰減：未設定正確的靈敏度因子 (Sensitivity Factor)。參考 Section 7 的補償公式。
  • 發射率設定錯誤：測量金屬或反光表面時，需降低發射率參數（暫存器 0xCA）。
  • 視窗髒汙或結露：清潔保護窗表面，確保無遮蔽物。
Q: SPI 讀取的資料全為零或亂碼，如何排查？ ▼

常見通訊問題排查步驟：

  1. 確認 DATA_READY 接腳已拉高後才讀取。
  2. 檢查 SPI 模式是否為 Mode 0（CPOL=0, CPHA=0）。
  3. 確認 I2C 已先寫入 0xB1 啟動串流模式。
  4. 檢查時脈頻率是否超過感測器規格上限。
Q: 如何判斷感測器是否需要重新校正？ ▼

以下情況建議重新進行黑體校正：

  • 更換或加裝保護視窗後。
  • 長時間運作後出現明顯溫度漂移。
  • 機構設計變更導致感測器熱環境改變。
  • 產品出貨前的品質驗證階段。

高精度的熱成像應用不僅依賴優秀的感測器，更取決於對物理熱學的尊重與嚴謹的系統校正。
透過掌握視窗補償係數、實施嚴格的熱隔離設計，並善用處理晶片的濾波能力，開發者將能克服環境挑戰，實現可靠的輻射測溫產品。
