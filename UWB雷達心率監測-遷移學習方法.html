📡 UWB 雷達心率監測：一種遷移學習方法

利用 FMCW 雷達資料集加速 IR-UWB 系統的模型開發

🎯 核心摘要

雷達技術為消費電子產品（如手機）提供了連續、非接觸式的心率監測潛力。然而，每種雷達系統都需要龐大的配對資料集進行訓練。

本研究展示了在 FMCW（調頻連續波）和 IR-UWB（脈衝無線電超寬頻）雷達系統之間的遷移學習。研究提出一種新穎的 2D + 1D ResNet 架構，在 FMCW 雷達上達成了 0.85 bpm 的平均絕對誤差（MAE）。

關鍵的是，將此模型在小型的 IR-UWB 資料集上進行微調後，達到了 4.1 bpm 的 MAE（符合 ANSI/AAMI <5 bpm 標準），相較於 IR-UWB 基線模型 MAE 降低了 25%。這項成果證明了遷移學習的可行性，能大幅加速將雷達心率監測功能引入現有消費設備的進展。

📜 核心理論與背景

心率（HR）是關鍵的生理參數。雖然智慧手錶等穿戴式裝置可提供監測，但其採用率遠低於智慧型手機。透過手機等無處不在的設備實現非接觸式監測，將使更廣泛的人群受益。

雷達技術因其隱私保護、不受膚色影響、可穿透衣物等特性而成為首選。FMCW 和 IR-UWB 是最有潛力的兩種技術，且已逐漸整合到消費電子產品中。

主要的挑戰在於，心臟搏動引起的胸壁運動幅度僅為呼吸運動的 1/20。因此，演算法必須能準確區分心臟、呼吸及其他身體運動的訊號。

機器學習（ML）模型依賴大量高品質資料，為每個新的雷達配置收集資料既耗時又昂貴。本研究的核心目標是利用遷移學習，將一個大型資料集（FMCW）上學到的知識，轉移到一個資料集小得多的新系統（IR-UWB）上。

🛠️ 系統方法與詳細流程

🎯 研究目的：為何使用兩種雷達？

本研究的目的並非要同時使用兩種雷達，而是要解決一個關鍵的「資料困境」：

  • FMCW 雷達 (Soli): 已擁有一個大型、現有的高品質資料集（980 小時），但硬體較複雜，尚未普及到所有手機。
  • IR-UWB 雷達 (手機內建): 這是未來的目標應用（已內建於許多旗艦手機），但目前只有一個小型的專用資料集（37.3 小時）。

核心挑戰：如果直接用小型資料集訓練 UWB 模型，效果不佳（MAE 5.4 bpm）。因此，研究的核心是探索能否利用 FMCW 的「大型資料集」知識，來「轉移」並顯著提升 IR-UWB 上的「小型資料集」模型效能？這就是遷移學習（Transfer Learning）的價值所在。

🔍 特徵提取詳細流程 (訊號處理)

論文的詳細方法論分為訊號處理（特徵提取）和模型架構。訊號處理是將原始雷達反射訊號轉換為模型可用的「乾淨特徵」的關鍵步驟：

  1. 資料分段 (Windowing): 原始資料被切割成 60 秒的片段，並以 15 秒的步進 (Step) 產生重疊的樣本，以增加資料量。
  2. 速率統一 (Resampling):
    • FMCW (Soli): 原始 Chirps 先在 Burst 內取平均，達到 30 Hz。
    • IR-UWB: 原始 200 Hz 取樣率透過平均「降取樣」至 30 Hz，以統一模型的時間維度。
  3. 訊號轉換 (FFT): 對於 FMCW 雷達，對每個 30 Hz 的 Chirp (含 256 取樣點) 進行 FFT，轉換為 129 個有效「距離切片」(Range Bins)，構成「距離-時間圖」(Range Profile)。(IR-UWB 原始訊號已是此形式)。
  4. 靜態過濾 (Clutter Filter): 減去訊號的「時間平均值」，以消除來自房間內靜止物體（如桌椅、牆壁）的穩定反射，只保留動態訊號。
  5. 使用者偵測 (Presence Detection): 使用恆定誤報率 (CFAR) 演算法（閾值設為 1.5），在距離圖上動態找到最強的訊號峰值（即使用者）。若偵測不到，則該 60 秒片段將被丟棄。
  6. 關鍵特徵生成 (Key Features): 這是論文的關鍵貢獻。傳統方法多只用「幅度 (Magnitude)」，本研究同時提取了複數訊號的「幅度」和「解包裝後的相位 (Unwrapped Phase)」。相位對於偵測由心跳引起的 sub-mm 級微小運動極為敏感。
  7. 呼吸移除 (Adaptive Filter):
    • 首先，使用 3 Hz 高通濾波器 移除 <3 Hz 的低頻雜訊。
    • 接著，使用 Savitzky-Golay 濾波器（窗長 1.5s）估算出呼吸的訊號趨勢。
    • 最後將原始訊號減去這個呼吸趨勢。這能有效移除呼吸的主頻率及其諧波干擾。
  8. 模型輸入 (T×S×1): 特徵被送入模型前，會先進行 Min-Max 標準化 (Normalization)，並重塑 (Reshape) 為 T (時間) x S (空間特徵) x 1 (通道) 的張量，以便 2D 摺積核能同時捕捉空間相鄰特徵。

🧠 創新的 2D + 1D ResNet 架構

ResNet (殘差網路) 是一種深度神經網路，它使用「跳躍連接」(skip connections) 允許資訊直接跳過幾層，將原始輸入訊號「殘差」(residual) 添加到後續層的輸出上。這使得網路可以建得非常深，而不會因為梯度消失而「忘記」早期的重要特徵，非常適合處理複雜的模式。

本研究中 2D 與 1D 的維度定義非常關鍵：

  • 2D 維度 指的是：[維度1: 時間] x [維度2: 空間特徵]
  • 1D 維度 指的是：[維度1: 時間]

本研究設計的混合架構，每一步都有明確目的：

  1. 輸入層 (Input): 將所有特徵（例如：天線1的幅度、天線1的相位...等）在空間維度上「攤平」，排成一個 2D 矩陣 (時間 T x 空間特徵 S)。
  2. 2D ResNet (時空特徵): 模型首先像處理「圖像」一樣處理資料。它包含 2 個 Stage（Filters: 64, 128），Stride=2。它在 [時間] x [空間特徵] 的 2D 平面上進行摺積，同時學習時間和空間的關聯性。
  3. 平均池化 (Avg. Pooling): 在 2D ResNet 提取了豐富的時空特徵後，此步驟將「空間」維度進行壓縮（平均）。這就像是將所有空間特徵「總結」成一個單一的描述符，輸出變為一個 1D 的時間序列。
  4. 1D ResNet (時間特徵): 接著，一個非常深的 1D ResNet 專注於分析這個 [時間] 維度的 1D 序列。它包含 4 個 Stage（Filters: 128, 256, 512, 1024），Stride=2。它在更長的時間尺度上尋找複雜的節律和模式。
  5. 輸出層 (Output): 最終所有時間步的特徵被平均，形成一個 1024 維向量，再通過一個全連接層 (FC Layer) 估算出最終的心率值（BPM）。

⚔️ 本研究 vs. 傳統方法：關鍵差異

本研究的成功不僅來自遷移學習，更在於訊號處理與模型架構上，針對「傳統方法」的重大缺陷進行了改進：

1. 特徵選擇：「幅度」+「相位」

傳統方法：許多相關研究僅使用雷達訊號的「幅度」(Magnitude) 來估算心率，遺失了訊號的另一半資訊。

本研究方法：同時提取「幅度」與「解包裝後的相位」(Unwrapped Phase)。相位對 sub-mm 級微小運動極為敏感。
定量貢獻：消融實驗證實，兩者並用可使 MAE 降低達 65%。

2. 呼吸干擾：移除「諧波」

傳統方法：使用標準的「帶通濾波器」(Band-pass filters) 來濾除低頻的呼吸訊號。但這有重大缺陷：它們雖然能濾掉呼吸的「主頻率」，但無法濾除呼吸的「諧波」(Harmonics)。

例如：如果呼吸是 0.3 Hz，它的諧波 0.6 Hz、0.9 Hz... 就可能落入心率的範圍（例如 1.0 Hz / 60 bpm），造成嚴重干擾。

本研究方法：採用「自適應濾波器」(Adaptive Filter)，先估算出完整的呼吸趨勢（使用 Savitzky-Golay 濾波器），然後將其從原始訊號中「減去」。這能同時移除主頻率及其惱人的「諧波」。

3. 模型架構：「時空」vs.「效率」

傳統方法：多數模型（如 1D CNN 或 LSTMs）將多天線、多距離的訊號視為多個獨立「通道」(Channels) 來分析。這忽略了特徵之間在「空間」上的物理關聯性。

本研究方法：創新的 2D + 1D ResNet 架構。先用 2D ResNet 捕捉「時空」特徵，再用 1D ResNet 深入分析「時間」節律。
定量貢獻：此架構 (MAE 0.89 bpm) 遠勝於標準 1D ResNet (MAE 1.30 bpm)，且後者的參數量是本架構的 15 倍以上，突顯本架構的「效率優勢」。

🔄 遷移學習：實作流程與核心

遷移學習的目的：明確解決 IR-UWB 資料集過小的問題。如果直接在 37.3 小時的小資料集上訓練，模型效果不佳 (MAE 5.4 bpm)。因此，本研究希望利用 980 小時的 FMCW 資料集來「預先訓練」一個懂雷達心率的通用模型。

⚙️ 解決方案：詳細實作流程

  1. 步驟 1：資料「橋接」(Data Bridging) - 關鍵實作

    這是最關鍵的實作核心。目標是讓 FMCW 資料「偽裝」成 IR-UWB 的低解析度特性。

    • 根本挑戰：原始 FMCW 模型輸入的特徵維度極高：
      (3 個接收天線) x (32 個距離切片) x (2 個特徵 [幅+相]) = 192 個特徵。
      而目標 IR-UWB 模型最終僅使用：
      (1 個接收天線) x (1 個距離切片) x (2 個特徵 [幅+相]) = 2 個特徵。
    • 具體操作：研究人員故意「降級」大型 FMCW 資料集，僅從中選取「單一」天線 (1 antenna) 和「單一」距離切片 (1 range bin)，並額外加入高斯雜訊（σ=0.0005, p=0.7），以模擬 UWB 的低解析度與訊號品質。
  2. 步驟 2：模型預訓練 (Pre-training)

    使用這個 980 小時、但經過「降級」的 FMCW 資料集，來預先訓練 2D + 1D ResNet 模型。

    • 訓練目的：讓模型學會最困難的 95% 任務——如何從一個「低解析度、單點、高雜訊」的雷達訊號中，找出隱藏的心率特徵。
    • 成果：得到一個「低解析度雷達心率專家」模型（此模型在 FMCW 測試集上 MAE 為 2.45 bpm）。
  3. 步驟 3：模型微調 (Fine-tuning)

    將這個「低解析度專家」模型（已在 FMCW 資料上訓練完畢），拿來處理 37.3 小時的「真實 IR-UWB 資料集」。

    • 訓練目的：模型不再需要從頭學起，僅需「微調」參數，以適應 IR-UWB 訊號 5% 的微小特性差異。
    • 關鍵技巧：微調時，在 IR-UWB 資料上加入高斯雜訊 (p=0.6) 作為資料增強，並刻意交換「幅度」與「相位」的特徵順序，實驗證實這能進一步提升表現。

📊 實驗結果與分析

✅ FMCW SOTA 成果

使用 2D+1D ResNet 模型在完整 FMCW 資料集上 (3天線 x 32 bin)，達到了 0.85 bpm 的 MAE 和 1.42% 的 MAPE。

這超越了先前 1.69 bpm MAE 的 SOTA（最新技術水平），驗證了新模型架構與多特徵的優越性。

📉 IR-UWB 基線模型

僅使用小型 IR-UWB 資料集從頭訓練的模型，最終 MAE 為 5.4 bpm。

這個表現在臨床上處於可接受的邊緣，但準確度不穩定。

🚀 遷移學習的勝利

經過 FMCW 資料預訓練，再於 IR-UWB 資料上微調的模型，最終 MAE 達到了 4.1 bpm (MAPE 6.3%)

相較於 5.4 bpm 的基線，MAE 顯著降低了 25%，並且完全符合 ANSI/AAMI (<5 bpm) 的臨床準確度要求。

🔬 子群分析 & 失效模式

1. 資料量邊際效益

消融實驗顯示，微調時僅使用 40% 的 IR-UWB 訓練資料，就已帶來絕大多數的效能提升。這表明未來在收集新資料時，可以顯著節省成本。

2. 失效模式：FMCW (睡眠場景)

FMCW 模型的誤差主要來自「存在偵測 (CFAR)」的失敗。當使用者距離過近 (<0.5m) 或過遠 (> 1.5m)，或在「坐起」時，CFAR 演算法可能選錯 range bin，導致 MAE 瞬間飆高。這表明 CFAR 是系統的瓶頸。

3. 場景敏感性：IR-UWB (坐姿場景)

IR-UWB 模型的表現在不同位置（桌上 vs. 腿上）和不同站點之間基本一致。但由於訓練資料的局限，模型在高心率區間 (>90 bpm) 的效能會顯著下降。此外，在疾病、年齡、性別之間未觀察到顯著差異。

🏁 結論與未來展望

本研究不僅提出了一種創新的 2D + 1D ResNet 模型架構，在 FMCW 雷達上實現了 SOTA 的心率檢測準確性，更重要的是，首次成功展示了從高解析度 FMCW 雷達到低解析度 IR-UWB 雷達的遷移學習。

這項成果證實，我們無需為每一種新的雷達硬體都收集龐大的資料集。通過利用現有的大型資料集進行預訓練，可以大幅加速模型在新型或資源受限（如手機 UWB 晶片）硬體上的部署，為非接觸式心率監測技術的普及鋪平了道路。

🚧 限制與未來工作

  • 理想化場景： IR-UWB 資料是在理想化的實驗室條件下收集的，缺乏真實世界中常見的干擾。
  • 缺乏動作資料： 資料均為靜態（睡眠或坐姿），模型在運動、強動作下的表現尚未驗證。
  • 高心率樣本不足： 缺乏高心率樣本（如運動後）是導致模型在高 HR 區間效能下降的主因。
  • 單人場景： 尚未驗證在多人環境中（例如區分父母和孩子）的效能。
  • 硬體整合： 本研究使用評估板，未來需驗證模型在真正內建於手機的 UWB 晶片上的實際表現。
