第一章

合規為王：不可逾越的紅線

在中國內地，任何技術措施都必須以法律合規為前提。核心是 ICP 備案 與 警察備案，它們共同將您的真實身分與域名、伺服器 IP 進行了牢固綁定。

⚠️

罰則摘要

「未履行備案即提供服務」的法定處罰為 責令限期改正並處 ¥10,000 罰鍰；對「超出備案內容」或首頁未按規定懸掛/鏈向的違規，處 ¥5,000–¥10,000。拒不改正者可被關閉網站。

⏱️

時程節點

流程節點包括接入商初審、人臉核驗/簡訊核驗（24 小時內完成，逾期退回）以及省管局終審（法規時限最長 20 個工作日）。

👤 個人備案 vs. 企業備案：關鍵抉擇

個人備案原則上限於非經營性內容；如涉及留言、論壇、社區等互動功能，多數省份要求前置審批/會商或改以單位（企業）主體辦理，否則難以通過。

屬性個人備案企業備案
允許內容僅限個人作品、日誌等非商業性內容分享。可包含企業宣傳、產品推廣等，須符合經營範圍。
互動功能嚴格限制，部分省份（如江蘇）明令禁止使用者登入或留言板，或需前置審批。允許,但須承擔內容審核的法律責任。
主要文件個人身分證；非本地戶籍者通常需提供居住證。企業營業執照、法定代表人及網站負責人身分證。

🗺️ 備案規則的地區差異

  • 📍 江蘇：個人網站命名普遍要求採「姓名 + 的個人主頁/個人部落格」格式；涉及論壇需提供網信辦會商結果。
  • 📍 山西：個人備案受理條件嚴、域名數量與單次申報有限制，且居住證一般不被接受（以當期公告為準）。

🚨 關鍵合規：警察備案與首頁標註

內地託管網站上線時，首頁底部需標示 ICP 備案號並鏈向工信部備案管理系統指定網址；上線後30 日內完成警察機關聯網備案。違反可處 ¥5,000–¥10,000（首頁標註）或被警告/停機整頓（警察備案）。

第二章

技術加固：最小化暴露面

強化伺服器安全是降低風險的基礎。目標是在無差別的自動化掃描中，不給攻擊者和掃描器過多有用資訊。

🔐 基礎網路環境強化

  • 🔑 更改預設連接埠： 避開 SSH(22)、NAS管理(5000/5001)等熟知連接埠。
  • 🛑 停用閒置服務： 關閉 Telnet, FTP 等不必要的服務。
  • 🔒 啟用MFA與自動封鎖： 為管理帳戶啟用多因素驗證，並設定自動封鎖多次登入失敗的IP。
  • 🚪 確認連接埠策略： 在公有雲環境普遍封鎖 25/TCP 出站；住宅寬頻對 135/139/445/25 等連接埠的策略因電信業者而異。

🤫 Nginx 靜默防禦：return 444

return 444 的效果是關閉連線且不返回 HTTP 負載，可降低指紋暴露；但掃描端仍可觀測到 TCP 已開放並被主動關閉，並非與「未開放連接埠」不可區分。

第三章

進階防禦：理解對手的工具

對於境外服務，您面臨的是純粹的技術對抗。高位隨機連接埠僅能降低泛用掃描噪音；對 ZMap/Masscan 等網際全網掃描無實質防禦力。理解 GFW 的偵測機制是規避封鎖的關鍵。

🕵️ SNI 洩露與流量分析

在未廣泛部署 ECH (Encrypted Client Hello) 的情況下，TLS 握手中的 SNI 仍為明文，可被用於域名級阻斷。此外，中國曾對 ESNI 施行攔截，且有針對 QUIC 的 SNI 攔截量測結果。

🧠 GFW 的高級流量分析：「負向過濾」模型

對於「全加密」協定的封鎖，GFW 存在三類基於可列印 ASCII 字元的豁免啟發式規則，已有公開量測與演算法重建。這意味著，純粹隨機、高熵的流量反而更容易被攔截。

📝
豁免規則 1 (Ex4)

若首 6 個位元組均為可列印 ASCII 字元，則豁免。

📊
豁免規則 2 (Ex2)

若封包中超過 50% 為可列印 ASCII 字元，則豁免。

📏
豁免規則 3 (Ex3)

若存在長度 >20 的連續可列印 ASCII 字串，則豁免。

在防火牆內，合規是強制性的；在防火牆外，模仿是必要的。您的目標不是變得不可見，而是要讓流量特徵「看起來很普通」，以匹配 GFW 的豁免規則。
