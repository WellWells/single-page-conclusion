記憶體運作原理與指標基礎

要理解指標，首先必須理解電腦記憶體（RAM）的運作方式。我們可以將記憶體視為一長串連續的「位元組（Byte）」序列。每個 Byte 都有一個獨一無二的位址（Address）（例如：0x7ff...）。

變數宣告的底層行為

當我們在程式中宣告變數時，編譯器會根據資料型別分配特定大小的記憶體空間。注意：int 的大小取決於編譯器與系統架構（Implementation-defined），雖然在現代系統常見為 4 Bytes，但標準僅保證 sizeof(short) ≤ sizeof(int) ≤ sizeof(long)。

指標（Pointer）本質上就是一個變數，但它儲存的不是數值，而是另一個變數的記憶體位址。 

// C指標基礎
int a = 5;
int *p = &a;   // p 儲存 a 的位址

// ✅ 正確輸出指標位址的方式：使用 %p 並轉型為 void*
printf("p = %p\n", (void*)p); 
printf("*p = %d\n", *p); // 輸出值：5 (Dereferencing)

指標型別強轉與運算邏輯

指標是強型別（Strongly Typed）的。這決定了兩件事： 1. 解參考時讀取的範圍：int* 讀取 sizeof(int)，char* 讀取 1 byte。 2. 指標運算的步長：p + 1 並非單純加 1，而是加上指向型別的大小。

💡 深度範例：Little Endian 下的記憶體剖析

假設 `int a = 1025` (4 bytes)，二進位為 `00000000 00000000 00000100 00000001`。 若將其強制轉型為 `unsigned char*` 逐位元組讀取： 

int a = 1025;
int *p = &a;
// 使用 unsigned char 避免有號數負值問題
unsigned char *c0 = (unsigned char*)p; 

printf("*c0 = %u\n", *c0);       // 1 (00000001)
printf("*(c0+1) = %u\n", *(c0+1)); // 4 (00000100)

注意：這展示了 Little Endian (低位元組存於低位址) 的特性。

Void 指標 (Generic Pointer) 限制

void *p 可以儲存任何位址。但標準 C 不允許 對 void* 進行算術運算 (如 p+1) 或解參考 (*p)，因為編譯器不知道大小。雖然 GCC 等編譯器有擴充支援 (視為 1 byte)，但這是非標準行為，應避免使用。

多重指標 (Pointer to Pointer)

既然指標變數本身也佔用記憶體空間並擁有位址，我們自然可以建立「指向指標的指標」。

  • int x = 5; (值)
  • int *p = &x; (指向 x)
  • int **q = &p; (指向 p)
  • int ***r = &q; (指向 q)

透過多重解參考 ***r，我們可以一路追溯回原始變數 x 的值。這在動態配置多維陣列或需要修改函式參數中的指標本身時非常有用。 

// 記憶體鏈結示意
// r(215) -> q(210) -> p(205) -> x(200)

*p  == 5
*q  == p (x 的位址)
**q == 5   (x 的值)
***r == 5  (x 的值)

函式傳參：Pass by Value 與指標模擬

嚴格來說，C 語言只有 Pass by Value (傳值)。當我們說「使用指標傳址」時，實際上是將變數的「位址值」複製給函式。

一般傳值

複製變數的值。函式內的 a 是副本，修改不影響外部。 

void Increment(int a) {
    a = a + 1; // 僅修改區域副本
}

指標傳遞 (模擬傳址)

複製變數的「位址」。透過解參考 (*p) 間接修改原始記憶體。 

void Increment(int *p) {
    *p = *p + 1; // 修改原始變數
}

陣列與指標的退化 (Decay)

在大多數表達式中，陣列名稱會退化 (Decay) 為指向首元素的指標。A[i] 僅僅是 *(A + i) 的語法糖。

陷阱：陣列參數的 Sizeof

當陣列傳遞給函式時，它完全變成了一個指標。因此在函式內 sizeof(A) 回傳的是指標大小 (通常 4 或 8 bytes)，而非陣列長度。 

void Func(int A[]) { // A 實際上是 int*
    // ❌ 錯誤：無法在此計算陣列長度
    // ✅ 正確：必須由呼叫者傳遞 size_t n
}

字串常數與未定義行為 (UB)

1. String Literals 的記憶體位置

字串字面值 (如 "Hello") 通常存放在 **Read-only Data Segment (唯讀資料區)**。

// Stack Array
char C[] = "Hello";
複製字串到 Stack 變數。
C[0] = 'A'; ✅ 合法
// Pointer to Literal
char *C = "Hello";
C 指向唯讀記憶體區段。
C[0] = 'A'; ⚠️ Undefined Behavior
(常見結果為 Segmentation Fault 崩潰)

多維陣列的記憶體幾何

對於 int B[2][3]，記憶體是線性排列的。

多維陣列通用指標公式

B[i][j] = *(*(B + i) + j)
  • B：型別為 int (*)[3] (指向含有3個整數的陣列)。
  • *B：型別為 int* (指向整數)。
  • B + 1：跳過 3 個整數 (12 bytes)。
  • *B + 1：跳過 1 個整數 (4 bytes)。

C 語言動態記憶體 (Heap)

malloc(size) 配置指定 Bytes，內容未初始化 (Garbage)。
calloc(n, size) 配置並將內容初始化為 0。
realloc(ptr, size) 調整大小，可能移動資料位址。
free(ptr) 釋放記憶體。忘記會導致 Leak。

危險操作：指標作為回傳值

🚫 嚴重錯誤

絕對不要回傳 Stack 區域變數的位址。函式結束後，Stack Frame 被銷毀，該位址指向的記憶體隨時會被覆蓋 (Dangling Pointer)。

int* Add(int a, int b) {
    int c = a + b;
    return &c; // ❌ 危險！c 已死亡
}

✅ 正確做法

若需回傳資料，應將其配置於 Heap (需手動 free) 或由呼叫者提供緩衝區。

int* Add(int a, int b) {
    int *c = (int*)malloc(sizeof(int));
    *c = a + b;
    return c; // ✅ Heap 直到 free 前都存在
}

函式指標與 Callback (qsort 實戰)

利用函式指標，我們可以將「行為」作為參數傳遞。

qsort 比較函式的陷阱

很多教學使用 `return A - B`，這在 `A` 為大正數、`B` 為大負數時可能造成整數溢位 (Integer Overflow)，導致排序錯誤。 

// ✅ 安全的三向比較實作
int compare(const void *a, const void *b) {
    int A = *(const int*)a;
    int B = *(const int*)b;
    // 回傳 (A > B) - (A < B) 確保回傳 1, 0, 或 -1
    return (A > B) - (A < B);
}

qsort(A, n, sizeof(int), compare);

記憶體洩漏 (Memory Leak)

當 Heap 記憶體被配置 (malloc/new) 卻失去所有指向它的指標且未被釋放 (free/delete) 時，就發生了 Memory Leak。這不會立即導致 Crash，但會導致程式佔用的記憶體不斷增加，最終可能導致系統強制終止程式 (OOM Kill)。

賭博遊戲範例：若每次 Play() 都 malloc 新陣列卻沒 free，玩 1000 局後將有 1000 個無法存取的陣列滯留於 Heap 中。

C++ 現代 C++ 記憶體管理 (Modern C++)

在現代 C++ (C++11 以後)，我們強烈建議避免使用 raw pointers (裸指標) 與 malloc/free 來管理資源。應採用 RAII (Resource Acquisition Is Initialization) 模式與 Smart Pointers (智慧指標)。

智慧指標 (Smart Pointers)

  • std::unique_ptr

    獨占擁有權。超出作用域 (Scope) 時自動釋放記憶體。零成本抽象 (Zero-overhead)。

  • std::shared_ptr

    共享擁有權。透過 Reference Counting 計數，計數歸零時釋放。

引用 (Reference) vs 指標

C++ 的 Reference (int &ref = a;) 更安全且保證非空 (Non-null)。 

// C++ 推薦寫法
void Increment(int &a) { // Pass by Reference
    a++; 
}
// 呼叫端更簡潔：Increment(x);

現代容器取代原生陣列

使用 std::vector 或 std::array 取代 int arr[] 或 malloc。它們會自動管理記憶體，且知道自己的大小 (.size())，避免了 C 語言陣列退化與洩漏的風險。

結語

指標是 C/C++ 的雙面刃。在 C 語言中，它是操作底層效能的唯一途徑，需嚴謹遵守標準並防範 UB；而在現代 C++ 中，我們應善用 RAII 與智慧指標來封裝複雜的記憶體管理，讓程式碼更安全、更強健。
