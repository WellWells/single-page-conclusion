分析摘要

此攻擊展現了極高的專業度，攻擊者不僅具備帳號滲透能力，還開發了針對不同 OS (Win/Mac/Linux) 的客製化二進制後門，目標在於獲取財務利益與敏感資訊。

建議立即審核所有 CI/CD 流程

事故發生原因：權限失控與帳號劫持

1. 維護者帳號遭劫持 (Account Takeover)

根據 GTIG 的調查，本次攻擊的起點在於 axios 套件其中一位具備發布權限的維護者帳號遭到滲透。攻擊者成功更改了與該 NPM 帳號關聯的電子郵件地址，並將其指向攻擊者控制的帳戶： ifstap@proton.me

*註：這顯示即便有 MFA，若攻擊者能透過電子郵件找回機制或特定 Session 劫持手段掌握帳號，仍能發布惡意更新。

2. 注入惡意依賴項 (Transitive Dependency Injection)

攻擊者並未直接修改 Axios 的主程式碼，而是利用其發布權限，將一個名為 plain-crypto-js 的惡意包加入到 Axios 的依賴清單中。

這種做法更具隱蔽性，因為許多自動化安全掃描器會檢查主包的原始碼變更，但往往會忽略新加入的、看似合理的「加密混淆」輔助包。

攻擊生命週期解析

1

觸發：Postinstall Hook

當開發者執行 npm install 時，NPM 會自動觸發 postinstall 腳本。攻擊者在 plain-crypto-js 中定義了如下動作：

"scripts": {
  "postinstall": "node setup.js"
}

惡意腳本 setup.js 會在背景靜默執行，開發者在終端機通常看不到異常訊息。

2

Payload 分發策略

偵測作業系統： 使用 Node.js 內建的 os.platform() 進行精準判讀。
自毀與清理： 執行完畢後會刪除 setup.js 並將 package.json 恢復原樣以隱藏蹤跡。
二階段下載： 透過 curl 向 C2 請求不同路徑的二進制檔案（如：product0/1/2）。

工程師防禦與修復指南

若您目前正在開發 Node.js 專案，請立即執行以下步驟：

1. 強制降級版本

避免使用受感染版本，並鎖定已知安全版本。

npm install axios@1.14.0

2. 審計依賴樹

檢查是否間接引入了惡意包 plain-crypto-js。

npm list plain-crypto-js

3. 清理全域快取

清除可能殘留惡意邏輯的 NPM 本地快取。

npm cache clean --force

重要提醒： 供應鏈攻擊的威力在於其「隱蔽性」。若確認曾安裝受影響版本，建議不僅要修復專案，還應對開發主機進行系統重設，並更換所有 API Token 與專案中的敏感金鑰 (Secrets)。
