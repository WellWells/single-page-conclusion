GNU/Linux Architecture

解構 GNU/Linux 系統元件

深入剖析 GNU/Linux 運作的核心：從啟動載入器、核心、初始程序到程序管理的技術概覽。

📌 核心元件摘要

一個 GNU/Linux 作業系統軟體主要由四個元件構成：Bootloader（啟動載入器）、Kernel（核心）、Shared Libraries（共享函式庫）以及 System Commands/Utilities（系統指令與工具）。本頁將詳細探討這些奠定系統運作基礎的元件，深入它們各自的角色與協同運作機制。

🚀 啟動載入器 (Bootloader)

電腦開機後，在韌體（BIOS）完成開機自檢（POST）後，啟動載入器是第一個執行的軟體。其主要職責是將作業系統核心載入到記憶體中。

由於啟動磁區（如 MBR）的空間限制（僅 440 bytes），啟動載入器通常採多階段設計。第一階段載入器位於 MBR，它的唯一任務就是載入位於檔案系統中的第二階段載入器，第二階段載入器再負責載入核心。

  • MBR (Master Boot Record): 位於已分割磁碟的啟動磁區。
  • VBR (Volume Boot Record): 位於未分割磁碟或特定分割區的啟動磁區。

🧠 系統核心 (Kernel)

核心運作於硬體與使用者空間（User-space）應用程式之間，是系統的核心。它負責管理所有硬體（CPU、記憶體、裝置）、執行使用者程式，並強制執行系統安全與完整性。

核心映像檔（如 zImage 或 bzImage）被載入後，其未壓縮的標頭部分首先在「真實模式」下執行，完成初步初始化後切換到「保護模式」，接著解壓縮核心主體並將控制權轉交過去。

核心最終會建立第一個使用者程序（PID 1），即 init 程序。核心功能可透過可載入模組 (Loadable Modules) 動態擴展，這在嵌入式系統中特別有用，可將驅動程式建置於核心內以節省記憶體。

📚 共享函式庫 (Shared Libraries)

共享函式庫是物件碼的集合，可在編譯時（靜態函式庫）或執行時（動態函式庫）連結到程式中。

使用共享函式庫的好處：

  • 可修改函式庫程式碼而無需重新編譯使用它的應用程式。
  • 減少記憶體使用：對於動態函式庫，記憶體中僅需保留一份副本，即可供多個執行中的程式共同使用。

🚦 初始程序 (Init Process)

PID 1，它是系統上所有其他程序的祖先程序。由核心啟動後，init 程序會接手啟動系統的各種背景服務（Daemons）。

傳統的 Sysvinit 使用「Runlevels」（執行等級）來定義系統狀態（如單使用者、多使用者、關機等）。不同的 Runlevel 會啟動或終止不同的服務。

現代系統多改用 systemd，它提供更強大但也更複雜的服務管理功能。但對於嵌入式系統，Sysvinit 的簡潔性仍有其優勢。

🗂️ 根檔案系統 (Root Filesystem)

GNU/Linux 系統至少需要一個掛載於 /（根目錄）的檔案系統。它包含了支援系統運作所需的大部分檔案。

檔案系統階層標準 (FHS) 定義了目錄結構的規範，使軟體和使用者能預測檔案位置。

常見檔案系統類型：

  • ext2/3/4: Linux 的標準檔案系統（Extent filesystem）。
  • SquashFS: 一種壓縮的唯讀檔案系統，常用於嵌入式系統。
  • XFS: 一種高效能的日誌型檔案系統。
  • NFS: 網路檔案系統（Network File System）。

FHS 主要目錄：

  • /bin: 供所有使用者使用的必要指令。
  • /boot: 啟動載入器檔案與核心。
  • /dev: 裝置檔案。
  • /etc: 系統主機特定的設定檔。
  • /lib: 共享函式庫與核心模組。
  • /sbin: 僅供系統管理員使用的必要系統二進位檔。
  • /proc: 虛擬檔案系統，提供行程與核心資訊。

⚙️ 行程管理 (Process Management)

行程是「執行中的程式」。系統使用 fork() 系統呼叫建立一個目前行程的複製品，然後使用 exec() 家族的系統呼叫將新建立的行程替換為新的程式映像。

Signals (訊號):

訊號是軟體中斷，用於通知行程發生特定事件（如 SIGINT, SIGKILL）。行程可選擇三種回應：

  • 執行預設動作（通常是終止）。
  • 忽略訊號。
  • 捕捉訊號並執行特定函式。

可使用 kill -l 查詢所有訊號，或使用 kill -[SIGNAL] [PID] 傳送訊號。

🕹️ 工作控制 (Job Control)

Shell 提供了工作控制功能，允許行程在前景或背景執行。

  • 前景 (Foreground): 行程會佔據終端機控制權，Shell 會等待它執行完畢。
  • 背景 (Background): 使用 & 運算子，行程會在背景執行，Shell 會立即返還提示符。
  • 暫停 (Suspend): 按下 Ctrl-Z 會傳送 SIGSTOP (或 SIGTSTP) 訊號，暫停前景行程。

↔️ 行程 I/O (Process I/O)

每個行程預設開啟三個 I/O 串流，並有對應的檔案描述符 (File Descriptor)：

  • Standard Input (stdin, 0): 預設為鍵盤。
  • Standard Output (stdout, 1): 預設為螢幕。
  • Standard Error (stderr, 2): 預設為螢幕。

I/O Redirection (重導向): 可使用 > (stdout), < (stdin), 2> (stderr), >> (附加) 將串流重導向至檔案。

Pipe (|): 一種 IPC 機制，可將一個行程的 stdout 連接到另一個行程的 stdin。

🌳 行程環境與 Shell 參數

Shell 層級

每個 Shell 行程都有一組 name=value 形式的變數。子行程只會繼承父行程中已匯出 (export) 的變數。

Quoting (引號): "雙引號" 允許變數擴展，'單引號' 則會阻止擴展，顯示字面內容。

Shell 特殊參數:

  • $1, $2...: 位置參數 (存取腳本參數)。
  • $#: 已分配的位置參數數量。
  • $?: 上一個指令的回傳碼 (0 代表成功)。
  • $$: 目前 Shell 的 PID。
  • $!: 最後一個背景行程的 PID。

C 語言層級 (Process Environment)

在程式碼層級，行程環境是可供存取的。可使用 C 函式庫中的 getenv() 來讀取，或使用 setenv()、putenv() 來設定。

範例 (getenv.c):

#include <stdio.h>
#include <stdlib.h>

int main(int argc, char **argv)
{
    char *value;

    if (argc < 2) {
        printf("用法: %s <變數名稱>\n", argv[0]);
        exit(-1);
    }

    value = getenv(argv[1]);

    if (value != NULL) {
        printf("變數 %s 的值是: %s\n", argv[1], value);
    } else {
        printf("找不到變數 %s\n", argv[1]);
    }
    return 0;
}

🖥️ 虛擬化技術 (Virtualization)

虛擬化技術允許在單台實體機上執行多個作業系統實例。Hypervisor（虛擬機監視器）是建立虛擬環境的軟體。

  • Type 1: 原生 (Native)，直接在硬體上執行 (如 VMware ESX)。
  • Type 2: 託管 (Hosted)，在一個宿主作業系統上執行 (如 VirtualBox, Qemu)。

User Mode Linux (UML) 是一種特殊的虛擬化，它將 Linux 核心編譯為一個標準的使用者行程，使其能在另一個 Linux 核心之上執行，無需硬體虛擬化支援。

Vagrant 是一種用於建立和管理虛擬機環境的工具，常與 Type 2 虛擬化軟體（如 VirtualBox）協同作業，以自動化設定和部署流程。

📋 Unix System V Runlevels (LSB)

ID 名稱 描述
0 Halt 系統關機。
1 Single-user mode 單使用者模式，用於管理任務。
2 Multi-user mode 多使用者模式，無網路。
3 Multi-user mode with networking 啟動系統（含網路）。
4 Not used/User-definable 使用者自定義。
5 Multi-user mode with GUI 同 Runlevel 3，但包含圖形介面管理器。
6 Reboot 重啟系統。

🏁 總結

GNU/Linux 是一個由多個精密元件構成的 layered system。從 Bootloader 喚醒硬體、Kernel 建立作業系統基礎，到 Init Process 啟動服務，再由檔案系統提供組織結構，最後透過強大的行程管理與 I/O 機制執行應用程式。理解這些核心元件的職責與互動，是掌握 Linux 系統運作的基石。
